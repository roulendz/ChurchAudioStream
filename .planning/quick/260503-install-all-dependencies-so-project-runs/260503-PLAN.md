# Install all dependencies so project runs

## Goal
Get ChurchAudioStream dev environment fully bootstrapped on this Win10 machine: Rust toolchain present, all 3 npm projects installed + built, Tauri dev launches and serves 7 live signals.

## Pre-flight (verified 2026-05-03)
- Node v24.15.0 + npm 11.12.1 — `C:\nvm4w\nodejs\`
- mingw-w64 gcc 16.1.0 — `C:\mingw64\bin\gcc.exe` (WinLibs UCRT)
- Git, GStreamer on PATH

## Tasks

### Task 1 — Rust toolchain (rustup + cargo + rustc)
- Download `rustup-init.exe` from win.rustup.rs
- Install with `--default-host x86_64-pc-windows-gnu --default-toolchain stable --profile minimal -y`
- Verify `rustc -vV` reports `host: x86_64-pc-windows-gnu`
- Smoke test: `cargo new --bin rust-smoke && cargo build`

### Task 2 — Node deps + project builds (build order: listener → sidecar → root)
- `cd listener && npm install && npm run build` → produces `sidecar/public/index.html`
- `cd sidecar && npm install && npm run build` → produces `src-tauri/binaries/server-x86_64-pc-windows-gnu.exe`
- `cd <root> && npm install` → installs Tauri CLI + admin UI deps

### Task 3 — Launch + verify 7 live signals
- Pre-kill orphan procs (churchaudiostream, server-*, mediasoup-worker, gst-launch-1.0, msedgewebview2)
- Launch `npm run tauri dev` in background
- Wait ~30s for cold Rust compile
- Verify processes: server-x86_64-pc-windows-gnu, mediasoup-worker, churchaudiostream, msedgewebview2
- Verify ports listening: 1420 (Vite), 7777 (HTTPS), 7778 (admin loopback)
- Freshness gate: server process StartTime ≥ binary LastWriteTime
- Log tail shows `=== SESSION START ===`, `Server listening on https://0.0.0.0:7777`, `Admin loopback listening on http://127.0.0.1:7778`

## Checkpoint
Halt for user "approved" once admin window visible + all 7 signals green.

## State observed at start
Most artifacts already present. Confirm state, skip redundant work, only run what's missing.
