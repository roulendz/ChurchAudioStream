## MAJOR

### MA-01: `should_check_now` debug_assert contradicts backward-clock branch
**File:** `src-tauri/src/update/checker.rs:26-35`
**Issue:** Line 26-29 asserts `now_unix >= last_check_unix` ("monotonic clock contract"), line 32 explicitly handles `now_unix < last_check_unix` as the backward-clock case. Debug builds panic on the exact case the function claims to handle. Test `should_check_when_clock_went_backward` (line 120) uses `now == last`, never exercises actual `now < last`. NTP correction backward in production = silent wrong, in `cargo test` = panic.
**Fix:**
```rust
pub fn should_check_now(
    last_check_unix: i64,
    now_unix: i64,
    min_interval_seconds: i64,
) -> bool {
    debug_assert!(min_interval_seconds >= 0);
    // No assertion on now_unix vs last_check_unix — backward skew handled below.

    if last_check_unix == 0 || now_unix < last_check_unix {
        return true;
    }
    now_unix.saturating_sub(last_check_unix) >= min_interval_seconds
}
```
Add a real backward-clock test: `assert_eq!(should_check_now(1_700_000_000, 1_699_999_000, 3_600), true);`

### MA-02: `save` is non-atomic — partial write corrupts state file
**File:** `src-tauri/src/update/storage.rs:67-71`
**Issue:** `std::fs::write(path, json)` truncates then writes. Crash / power loss / disk-full mid-write = truncated JSON. Next `load` returns `StorageError::Parse`. User loses dismiss/skip state permanently until file deleted manually. Doc comment line 66 "Single-syscall, non-atomic" calls out the problem but does not solve it. Phase 3 wires this into startup → first crash bricks update checks for that user.
**Fix:**
```rust
pub fn save(path: &Path, state: &UpdateState) -> Result<(), StorageError> {
    let json = serde_json::to_string_pretty(state)?;
    let tmp = path.with_extension("json.tmp");
    std::fs::write(&tmp, json)?;
    std::fs::rename(&tmp, path)?;
    Ok(())
}
```
Add test: write valid file, write a second time, kill before rename (simulate by writing tmp directly), call `load` → original survives.

### MA-03: corrupt JSON has no recovery path
**File:** `src-tauri/src/update/storage.rs:62`
**Issue:** `serde_json::from_str(&content)?` propagates `StorageError::Parse` on any malformed JSON (corrupt write, manual edit, disk error). Phase 3 caller has no signal "this is recoverable, just delete and reset to default". Update system permanently broken until user finds and deletes `update-state.json`. Tiger-Style fail-fast applies to programmer errors, not user-data corruption — the latter should self-heal.
**Fix:** Either (a) add a `load_or_default(path)` helper that logs parse errors and returns `UpdateState::default()`, or (b) document the contract: caller must catch `StorageError::Parse`, log, delete file, retry. Pick one — Phase 3 must not invent its own recovery.

### MA-04: payload field name diverges from decision variant — doc-comment lies
**File:** `src-tauri/src/update/dispatcher.rs:14, 17-21` vs `src-tauri/src/update/checker.rs:12-16`
**Issue:** Doc at dispatcher.rs:14 says "Field shape matches UpdateDecision::Notify". It does not.
- `UpdateDecision::Notify { version, notes, platform_url }`
- `UpdateAvailablePayload { version, notes, download_url }`
DRY violation across module boundary. Phase 3 will write `download_url: notify.platform_url` glue and the next reviewer will flag the asymmetry. Pick one name now.
**Fix:** Rename `UpdateDecision::Notify.platform_url` → `download_url` (callers: 0 outside the module, 4 test sites in checker.rs). Update tests. Re-export shape in mod.rs unchanged.

### MI-01: `should_check_now` integer underflow in pathological state
**File:** `src-tauri/src/update/checker.rs:35`
**Issue:** If `last_check_unix == i64::MIN` (corrupted state file) and `now_unix == 1`, the early return at line 32 doesn't fire (`1 > i64::MIN`), then `now_unix - last_check_unix` overflows. Debug builds panic, release wraps silently. Same fix as MA-01 (`saturating_sub`).
**Fix:** Same as MA-01.

### MI-02: `evaluate_update` underflow on backward-clock dismiss
**File:** `src-tauri/src/update/checker.rs:65`
**Issue:** `now_unix - last_dismissed_unix < dismiss_cooldown_seconds`. If user dismissed at T, then NTP corrects clock backward, `now_unix < last_dismissed_unix` → debug panic, release wraps to huge positive (cooldown false-passes — user gets re-notified, not a security issue but symmetric with MA-01).
**Fix:**
```rust
if now_unix.saturating_sub(last_dismissed_unix) < dismiss_cooldown_seconds {
    return UpdateDecision::SilentSkip("dismissed cooldown".to_string());
}
```

### MI-03: missing tests for decision-matrix edge cases
**File:** `src-tauri/src/update/checker.rs` test module
**Issue:** No coverage for:

- Boundary `now_unix - last_dismissed_unix == dismiss_cooldown_seconds` exactly (line 65 uses `<`, equal allows notify).

### MI-04: missing storage tests for failure modes
**File:** `src-tauri/src/update/storage.rs` test module
**Issue:** No coverage for:
- Corrupted JSON returns `StorageError::Parse` (write `"{not json"` to file, assert variant).
- `save` to path with non-existent parent dir returns `StorageError::Io`.
- `save` to read-only path returns `StorageError::Io` (Unix only — gate with `#[cfg(unix)]`).
- Round-trip preserves Unicode in `skipped_versions` (e.g. version strings are ASCII today but if pre-release tags ever land...).
**Fix:** Add 3 tests minimum (corrupted, missing parent, round-trip stability).

### MI-05: `Path` contract undocumented for Phase 3 async
**File:** `src-tauri/src/update/storage.rs:51, 67`
**Issue:** `load` and `save` are sync `std::fs`. Phase 3 background update-check task running on tokio runtime that calls these directly will block the runtime worker thread. No module-doc warning. Phase 3 author must remember `tokio::task::spawn_blocking`.
**Fix:** Add to module doc:
```
//! IO is sync (`std::fs`). Async callers must wrap calls in
//! `tokio::task::spawn_blocking` to avoid stalling the runtime.
```

### MI-06: DRY — duplicated linear search-by-version
**File:** `src-tauri/src/update/checker.rs:38-40` and `src-tauri/src/update/storage.rs:82`
**Issue:** Both implement `iter().any(|v| v == version)` against a `&[String]`. Two call sites, one helper would do.
**Fix:** Extract once. Either `pub fn contains_version(haystack: &[String], needle: &str) -> bool` in `version.rs` (logical home) or inline acceptance — pick one and stick to it. Today both call sites are 1-liners so the duplication cost is low; flag for re-evaluation if a third site appears.
