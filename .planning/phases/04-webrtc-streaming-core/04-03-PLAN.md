  <name>Task 1: Implement WorkerManager for mediasoup Worker lifecycle</name>
  <files>sidecar/src/streaming/worker-manager.ts</files>
  <action>
    Create `sidecar/src/streaming/worker-manager.ts` implementing WorkerManager class.
    Extends EventEmitter. Events: `worker-died`, `worker-restarted`, `workers-restarted`, `worker-memory-warning`, `worker-resource-update`.

    **Constructor:** Takes mediasoup config (workerCount, rtcMinPort, rtcMaxPort, logLevel) from AppConfig.

    **`start()` method:**
    - Create `workerCount` workers via `mediasoup.createWorker()` with config options
    - Store each in `WorkerState[]` with index and memoryMonitorInterval
    - Register `worker.on("died")` -> log, emit `"worker-died"`, call `restartWorker(index)`

    **`restartWorker(index)` method:**
    - Create new worker, replace old entry, start memory monitoring, emit `"worker-restarted"`

    **`getWorkerForChannel(channelId)` method:**
    - Round-robin: hash channelId to worker index (modulo)

    **`getWorkerResourceInfo()` method:**
    - Returns `WorkerResourceInfo[]` via `worker.getResourceUsage()` on each worker

    **`stop()` method:** Clear intervals, close all workers.

    **`restartAllWorkers()` method:** Close all, recreate, emit `"workers-restarted"`.

    **SRP note:** Memory monitoring is a separate concern. Implement via a private `WorkerMemoryMonitor` helper class within the same file (not exported). This helper:
    - Runs `worker.getResourceUsage()` every 60 seconds per worker
    - Logs warning at 500MB threshold (`WORKER_MEMORY_WARNING_THRESHOLD_KB = 512_000`)
    - Emits events via callback to WorkerManager (which re-emits `"worker-memory-warning"` and `"worker-resource-update"`)
    - If memory monitoring logic grows in Phase 8 (rotation), it can be extracted to its own file without changing WorkerManager's public API.
  </action>
  <verify>
    `npx tsc --noEmit -p sidecar/tsconfig.json` -- zero errors.
    WorkerManager importable: `npx tsx --eval "import { WorkerManager } from './sidecar/src/streaming/worker-manager.js'; console.log('OK');"`.
  </verify>
  <done>WorkerManager creates configurable workers with crash recovery and delegated memory monitoring.</done>
</task>

  <name>Task 2: Implement RouterManager for per-channel Router lifecycle</name>
  <files>sidecar/src/streaming/router-manager.ts</files>
  <action>
    Create `sidecar/src/streaming/router-manager.ts` implementing RouterManager class.

    **Constructor:** Takes WorkerManager and PlainTransportManager references.

    **`createChannelRouter(channelId, rtpPort, rtcpPort, ssrc)` method:**
    - Get worker via `workerManager.getWorkerForChannel(channelId)`
    - Create Router with Opus codec: `worker.createRouter({ mediaCodecs: [OPUS_RTP_CODEC] })`
    - Register `router.on("workerclose")` to clean up from tracking Map
    - Delegate PlainTransport + Producer to PlainTransportManager
    - Store full ChannelStreamingState in `Map<string, ChannelStreamingState>`

    **`removeChannelRouter(channelId)` method:** Close router (cascades to children), remove from Map.

    **Accessors:**
    - `getRouterForChannel(channelId)` -- Router or undefined
    - `getProducerForChannel(channelId)` -- Producer or undefined
    - `getActiveChannelIds()` -- channelIds with active producers
    - `getActiveChannelList(metadataResolver)` -- `ListenerChannelInfo[]` for listeners

    **`handleWorkerRestart(workerIndex)` method:**
    - Recreate routers/transports/producers for affected channels on new worker
    - Emit `"channel-streaming-restored"` per channel
    - Wire to WorkerManager's `"worker-restarted"` event
  </action>
