<task type="auto">
  <name>Task 1: Implement WorkerManager for mediasoup Worker lifecycle</name>
  <files>sidecar/src/streaming/worker-manager.ts</files>
  <action>
    Create `sidecar/src/streaming/worker-manager.ts` implementing WorkerManager class.

    **Constructor:** Takes mediasoup config (workerCount, rtcMinPort, rtcMaxPort, logLevel) from AppConfig.

    **`start()` method:**
    - Create `workerCount` mediasoup workers via `mediasoup.createWorker()`
    - Worker options: `logLevel` from config, `logTags: ["info", "ice", "dtls", "rtp", "rtcp"]`, `rtcMinPort` and `rtcMaxPort` from config
    - Store each worker in a `WorkerState[]` array with index and memoryMonitorInterval
    - For each worker, register `worker.on("died")` handler that:
      - Logs the error with worker index
      - Emits `"worker-died"` event with worker index and error
      - Calls `restartWorker(index)` to auto-restart (per locked decision: basic auto-restart on worker crash)

    **`restartWorker(index)` method:**
    - Create a new worker with same options
    - Replace the old WorkerState entry
    - Start memory monitoring for the new worker
    - Emit `"worker-restarted"` event with index
    - Note: Router/transport recreation is handled by RouterManager listening to this event

    **`startMemoryMonitoring()` private method:**
    - For each worker, run `worker.getResourceUsage()` every 60 seconds
    - Track `ru_maxrss` (peak RSS in kilobytes)
    - Log warning when memory exceeds 500MB threshold (per research: `WORKER_MEMORY_WARNING_THRESHOLD_KB = 512_000`)
    - Emit `"worker-memory-warning"` event with worker index, memoryMb, and threshold
    - Emit `"worker-resource-update"` event with WorkerResourceInfo for admin dashboard

    **`getWorkerForChannel(channelId)` method:**
    - Round-robin assignment: hash channelId to worker index (simple modulo)
    - Return the Worker from the WorkerState array

    **`getWorkerResourceInfo()` method:**
    - Returns `WorkerResourceInfo[]` for admin dashboard display
    - Calls `worker.getResourceUsage()` on each worker

    **`stop()` method:**
    - Clear all memory monitoring intervals
    - Close all workers via `worker.close()`

    **`restartAllWorkers()` method (for admin "restart workers" button):**
    - Close all workers, then recreate them
    - Emit `"workers-restarted"` event

    Extend EventEmitter for events: `worker-died`, `worker-restarted`, `workers-restarted`, `worker-memory-warning`, `worker-resource-update`.
  </action>
  <verify>
    `npx tsc --noEmit -p sidecar/tsconfig.json` compiles with no errors.
    WorkerManager can be imported: `npx tsx --eval "import { WorkerManager } from './sidecar/src/streaming/worker-manager.js'; console.log('WorkerManager OK');"`.
  </verify>
  <done>WorkerManager creates configurable number of mediasoup workers with crash recovery, memory monitoring, and round-robin channel assignment.</done>

  <name>Task 2: Implement RouterManager and PlainTransportManager</name>
  <files>

  <action>
    **Create `sidecar/src/streaming/router-manager.ts`:**

    RouterManager class manages per-channel Routers and coordinates with PlainTransportManager.

    **Constructor:** Takes WorkerManager and PlainTransportManager references.

    **`createChannelRouter(channelId, rtpPort, rtcpPort, ssrc)` method:**
    - Get worker via `workerManager.getWorkerForChannel(channelId)`
    - Create Router with Opus codec: `worker.createRouter({ mediaCodecs: [OPUS_RTP_CODEC] })`
    - Register `router.on("workerclose")` to clean up channel from tracking Map
    - Store as ChannelStreamingState in a `Map<string, ChannelStreamingState>`
    - Delegate PlainTransport + Producer creation to PlainTransportManager: `plainTransportManager.createForChannel(router, channelId, rtpPort, rtcpPort, ssrc)`
    - Store the PlainTransport and Producer in the ChannelStreamingState
    - Return the ChannelStreamingState

    **`removeChannelRouter(channelId)` method:**
    - Get ChannelStreamingState from Map
    - Close the router (cascades to transport, producer, consumers)
    - Remove from Map

    **`getRouterForChannel(channelId)` method:** Returns Router or undefined.
    **`getProducerForChannel(channelId)` method:** Returns Producer or undefined.
    **`getActiveChannelIds()` method:** Returns array of channelIds that have active producers.

    **`getActiveChannelList()` method:**
    - Returns `ListenerChannelInfo[]` for pushing to listeners
    - Takes a channel metadata resolver function to get name/language/description from AppChannel

    **`handleWorkerRestart(workerIndex)` method:**
    - Find all channels assigned to the restarted worker
    - Recreate their routers, PlainTransports, and producers on the new worker
    - Emit `"channel-streaming-restored"` event for each

    Wire to WorkerManager's `"worker-restarted"` event.

    **Create `sidecar/src/streaming/plain-transport-manager.ts`:**

    PlainTransportManager handles PlainTransport + Producer creation for GStreamer RTP ingestion.

    **`createForChannel(router, channelId, rtpPort, rtcpPort, ssrc)` method:**
    - Create PlainTransport with:
      ```
      listenInfo: { protocol: "udp", ip: "127.0.0.1", port: rtpPort }
      rtcpListenInfo: { protocol: "udp", ip: "127.0.0.1", port: rtcpPort }
      rtcpMux: false  // Matches Phase 3's separate RTP/RTCP ports
      comedia: true    // Auto-detect GStreamer source from first packet
      ```
    - Create Producer with:
      ```
      kind: "audio"
      rtpParameters.codecs: [{ mimeType: "audio/opus", clockRate: 48000, payloadType: OPUS_PAYLOAD_TYPE (101), channels: 2, parameters: { "sprop-stereo": 1 } }]
      rtpParameters.encodings: [{ ssrc }]
      ```
    - Register cleanup event handlers:
      - `plainTransport.on("routerclose")` -- clean up references
      - `audioProducer.on("transportclose")` -- clean up references
    - Return `{ plainTransport, audioProducer }`

    **KEY DESIGN:** PlainTransport persists across GStreamer restarts (per audit finding #3 and research anti-pattern). The transport listens on a fixed port; when GStreamer restarts and sends new RTP packets, comedia mode auto-detects the new source. No transport recreation needed.

    **`getTransportStats(channelId)` method:**
    - Returns PlainTransport stats for latency monitoring
