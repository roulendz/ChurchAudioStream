  <name>Task 1: Implement TransportManager for WebRtcTransport lifecycle</name>
  <files>sidecar/src/streaming/transport-manager.ts</files>
  <action>
    Create `sidecar/src/streaming/transport-manager.ts` implementing TransportManager class.

    **Constructor:** Takes `announcedIpAddress` (from `config.server.host`) -- the LAN IP used as `announcedAddress` in WebRtcTransport.

    **`createForListener(router, peerId)` method:**
    - Create WebRtcTransport with:
      ```
      listenInfos: [{ protocol: "udp", ip: "0.0.0.0", announcedAddress: this.announcedIpAddress }]
      enableUdp: true
      enableTcp: false    // UDP-only for lowest latency on local WiFi (locked: no TURN server)
      preferUdp: true
      initialAvailableOutgoingBitrate: 128_000  // Audio-only: 128 kbps Opus
      iceConsentTimeout: 30
      ```
    - Register event handlers:
      - `transport.on("icestatechange")` -- log state changes; if "disconnected" or "closed", log for monitoring
      - `transport.on("dtlsstatechange")` -- if "failed" or "closed", close transport and clean up
      - `transport.on("routerclose")` -- clean up from tracking Map
    - Store in `Map<string, mediasoupTypes.WebRtcTransport>` keyed by peerId
    - Return transport info: `{ id, iceParameters, iceCandidates, dtlsParameters }`

    **`connectTransport(peerId, dtlsParameters)` method:**
    - Get transport from Map, call `transport.connect({ dtlsParameters })`

    **`getTransport(peerId)` method:** Returns transport from Map.

    **`closeTransport(peerId)` method:**
    - Close transport, remove from Map

    **`getTransportStats(peerId)` method:**
    - Returns transport stats for per-listener connection info

    **`closeAll()` method:**
    - Close all transports, clear Map
  </action>
  <verify>
    `npx tsc --noEmit -p sidecar/tsconfig.json` compiles with no errors.
    TransportManager can be imported.
  </verify>
  <done>TransportManager creates on-demand WebRtcTransports per listener with LAN IP announced address, UDP-only, and proper event-driven cleanup.</done>

<task type="auto">
  <name>Task 2: Implement SignalingHandler for protoo request/notification handling</name>
  <files>sidecar/src/streaming/signaling-handler.ts</files>
  <action>
    Create `sidecar/src/streaming/signaling-handler.ts` implementing SignalingHandler class.

    **Constructor:** Takes RouterManager, TransportManager, and a channel metadata resolver function.

    **SRP note:** SignalingHandler focuses on dispatching protoo requests and managing consumers. Heartbeat detection and session tracking are delegated to private helpers within the file:
    - `PeerHeartbeatTracker` (private class) -- tracks last activity per peer, checks for zombies
    - Session info is derived from peer.data (ListenerPeerData) without a separate tracker

    **`handlePeer(peer)` method:**
    - Initialize peer.data with ListenerPeerData
    - Push active channels on connect via `peer.notify("activeChannels", ...)`
    - Register `peer.on("request", ...)` dispatching to handlers:

      **`getRouterRtpCapabilities`:** Return Opus capabilities, store client's rtpCapabilities in peer.data.

      **`createWebRtcTransport`:** Create via TransportManager, store in peer.data, accept with transport info.

      **`connectWebRtcTransport`:** Complete DTLS handshake via TransportManager.

      **`consume`:** Validate channel active + canConsume, close existing consumer if switching, create paused consumer, wire `producerclose`/`transportclose` events, accept with consumer params.

      **`resumeConsumer`:** Resume current consumer.

      **`switchChannel`:** Close current consumer, create new one on target channel. On failure: fall back to previous channel; if that also fails, notify with remaining active channels.

    - Register `peer.on("close")` -> close transport, emit `"listener-disconnected"`

    **Accessor methods:**
    - `getListenerCount(channelId?)` -- count peers excluding isAdmin
    - `getListenerSessions()` -- return ListenerSessionInfo[]
    - `notifyAllListeners(method, data)` / `notifyListenersOnChannel(channelId, method, data)`
  </action>
  <verify>
    `npx tsc --noEmit -p sidecar/tsconfig.json` -- zero errors.
    SignalingHandler importable.
  </verify>
  <done>Full protoo signaling flow: capabilities -> transport -> connect -> consume -> resume. Consumer always created paused. Channel switch with fallback.</done>

  <files>sidecar/src/ws/listener-handler.ts</files>
  <action>
    Create `sidecar/src/ws/listener-handler.ts` implementing ListenerWebSocketHandler class.

    **Constructor:** Takes HTTPS server, SignalingHandler, and streaming config (rateLimit, heartbeatInterval).

    **Setup:**
    - Create `protooServer.WebSocketServer(httpsServer, ...)` -- protoo wraps ws internally
    - Handle `connectionrequest`: validate `/ws/listener` path, apply rate limiting, accept, create peer, delegate to SignalingHandler

    **Rate limiting:**
    - Track connection attempts per IP in a Map with sliding window
    - Reject with 429 if IP exceeds `rateLimitPerIp` within `rateLimitWindowMs`

    **Heartbeat:**
    - Start interval from `config.streaming.heartbeatIntervalMs` (default 30s)
    - Delegate zombie detection to SignalingHandler's PeerHeartbeatTracker

    **`close()` method:** Close all connections and protoo WebSocketServer.

    **Path routing note:** protoo-server hooks into HTTP "upgrade" event. Since admin WebSocket also uses "upgrade", path-based routing must be handled in server.ts (see Plan 04-05, Task 2). This handler expects the upgrade to already be routed correctly.
  </action>
