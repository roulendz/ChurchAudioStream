  <name>Task 1: Create StreamingSubsystem facade</name>
  <files>sidecar/src/streaming/streaming-subsystem.ts</files>
  <action>
    Create `sidecar/src/streaming/streaming-subsystem.ts` implementing StreamingSubsystem class.

    **Pattern:** Follow AudioSubsystem facade pattern -- single entry point for all streaming functionality, extends EventEmitter.

    **Constructor:** Takes ConfigStore, AudioSubsystem reference.

    **`start(httpsServer)` method:**
    1. Read config for mediasoup and streaming settings
    2. Create WorkerManager with mediasoup config, call `workerManager.start()`
    3. Create PlainTransportManager
    4. Create RouterManager with WorkerManager and PlainTransportManager references
    5. Create TransportManager with `config.server.host` as announced IP
    6. Create SignalingHandler with RouterManager, TransportManager, and channel metadata resolver
    7. Create ListenerWebSocketHandler on httpsServer with SignalingHandler and streaming config
    8. Wire AudioSubsystem events:
       - `audioSubsystem.on("channel-state-changed", (channelId, status))`:
         - When status is "streaming": Create channel router + PlainTransport if not already active
           - Get channel from AudioSubsystem for processing config (rtpPort, rtcpPort, ssrc)
           - Call `routerManager.createChannelRouter(channelId, rtpPort, rtcpPort, ssrc)`
           - Notify listeners: `signalingHandler.notifyAllListeners("channelStateChanged", { channelId, state: "active" })`
           - Push updated active channel list to all listeners
         - When status is "stopped" or "error" or "crashed":
           - Notify listeners on that channel: `signalingHandler.notifyListenersOnChannel(channelId, "consumerClosed", { reason: "channel-stopped" })`
           - Notify all listeners of updated channel list
           - Remove channel router: `routerManager.removeChannelRouter(channelId)`
       - `audioSubsystem.on("channel-removed", (channelId))`:
         - Same as stopped: notify listeners, remove router
       - `audioSubsystem.on("channel-created", (channel))`:
         - Notify listeners of new channel available (per locked decision: push "new channel available" notification)
    9. Wire WorkerManager events:
       - `workerManager.on("worker-died", ...)` -- log alert, emit for admin dashboard
       - `workerManager.on("worker-memory-warning", ...)` -- emit for admin dashboard
    10. Log streaming subsystem started

    **`stop()` method -- Graceful shutdown (per locked decision: specific order):**
    1. Notify all listeners: `signalingHandler.notifyAllListeners("serverShuttingDown", { reason: "Server shutting down" })`
    2. Wait drain period (config.streaming.shutdownDrainMs, default 5s) -- let clients receive the notification
    3. Close ListenerWebSocketHandler (disconnect all listeners)
    4. Close TransportManager (close all WebRtcTransports)
    5. Close RouterManager (close all routers, cascading to PlainTransports and producers)
    6. Close WorkerManager (close all workers)
    7. Log streaming subsystem stopped

    **Delegated accessors for admin dashboard:**
    - `getListenerCount(channelId?)` -- delegates to SignalingHandler
    - `getListenerSessions()` -- delegates to SignalingHandler
    - `getWorkerResourceInfo()` -- delegates to WorkerManager
    - `getActiveStreamingChannels()` -- delegates to RouterManager

    **Events emitted:**
    - `"listener-count-changed"` (channelId, count) -- for admin dashboard broadcast
    - `"worker-alert"` (type, details) -- for admin dashboard alerts
  </action>
