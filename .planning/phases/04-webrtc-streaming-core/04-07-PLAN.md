  <name>Task 1: Fix metering parser regex to handle GValueArray format</name>
  <files>sidecar/src/audio/pipeline/metering-parser.ts</files>
  <action>
Update the metering parser to handle GStreamer 1.26's `(GValueArray)< value >` output format
in addition to the existing `(double)value` format.

**Actual GStreamer 1.26 output** (from live UAT testing):
```
level, endtime=(guint64)270630000000, timestamp=(guint64)270530000000, stream-time=(guint64)270530000000, running-time=(guint64)270530000000, duration=(guint64)100000000, rms=(GValueArray)< -104.47323284285487 >, peak=(GValueArray)< -90.308998699194362 >, decay=(GValueArray)< -90.308998699194362 >;
```

Multi-channel GValueArray format:
```
peak=(GValueArray)< -12.5, -14.2 >
```

Current format still supported (backward compatibility):
```
peak=(double)-12.5
peak=(double){ -12.5, -14.2 }
```

Steps:

1. Replace `buildFieldPattern` with a new function that matches BOTH formats:
   - `(double)` format: `peak=(double)-12.5` or `peak=(double){ -12.5, -14.2 }`
   - `(GValueArray)` format: `peak=(GValueArray)< -90.308 >` or `peak=(GValueArray)< -12.5, -14.2 >`
   The simplest approach: use a regex that matches `fieldName=(<type>)` then captures the values
   regardless of delimiter style (braces or angle brackets).

   Recommended regex strategy for `buildFieldPattern(fieldName)`:
   ```
   new RegExp(
     `${fieldName}=\\([^)]+\\)[{<]?\\s*([^;>}]+?)\\s*[>}]?(?:,\\s*(?:\\w+=|$)|;|$)`
   )
   ```
   This matches:
   - `fieldName=(anything)` -- handles both `(double)` and `(GValueArray)` type annotations
   - `[{<]?` -- optional opening brace or angle bracket
   - `([^;>}]+?)` -- captures the comma-separated values (lazy, stops at delimiters)
   - `[>}]?` -- optional closing angle bracket or brace
   - Terminated by comma+next-field, semicolon, or end-of-string

   IMPORTANT: Test the regex mentally against these cases before implementing:
   - `peak=(double)-12.5,` (single double, no brackets) -- captured group: `-12.5`
   - `peak=(double){ -12.5, -14.2 },` (double with braces) -- captured group: `-12.5, -14.2`
   - `peak=(GValueArray)< -90.308 >,` (GValueArray single) -- captured group: `-90.308`
   - `peak=(GValueArray)< -12.5, -14.2 >,` (GValueArray multi) -- captured group: `-12.5, -14.2`
   - `peak=(double)-inf;` (infinity) -- captured group: `-inf`

   If the recommended regex does not cleanly handle the single-double-no-bracket case
   (where there are no delimiters `{<>}`), consider a two-pattern approach instead:
   - Pattern A for `(double)` format (existing logic, works)
   - Pattern B for `(GValueArray)< ... >` format (new)
   - `buildFieldPattern` returns both, `parseMeteringLine` tries Pattern B first, falls back to Pattern A

2. Update the module-level doc comment (lines 11-13) to document the GValueArray format:
   ```
   * Handles multiple GStreamer output formats:
   *   Double single:     peak=(double)-12.5
   *   Double multi:      peak=(double){ -12.5, -14.2 }
   *   GValueArray single: peak=(GValueArray)< -90.308 >
   *   GValueArray multi:  peak=(GValueArray)< -12.5, -14.2 >
   ```

3. Rename `createStderrLineParser` to `createBusMessageLineParser` since it now processes
   stdout bus messages, not stderr. Update the JSDoc to reflect that it processes
   `gst-launch-1.0 -m` bus message output from stdout. Keep the function signature identical
   (same params, same return type). This is a rename for accuracy, not a behavior change.

4. Do NOT change `parseDoubleList`, `detectClipping`, `dbToNormalized`, or `parseMeteringLine`
   logic beyond the regex update. The parsing pipeline is correct once the regex captures the
   right values.

   - On match, emit a PipelineError with code "GSTREAMER_ERROR"
   - Non-error stderr lines get logged as debug (GStreamer often sends state change info to stderr)

   Implementation:
   ```typescript
   private attachStderrErrorDetector(child: ChildProcess): void {
     if (!child.stderr) return;

     const GSTREAMER_ERROR_PATTERN = /\b(?:ERROR|WARN|WARNING|CRITICAL)\b/i;
     let partialLine = "";

     child.stderr.on("data", (chunk: Buffer) => {
       const text = partialLine + chunk.toString("utf-8");
       const lines = text.split("\n");
       partialLine = lines.pop() ?? "";

       for (const rawLine of lines) {
         const line = rawLine.trimEnd();
         if (line.length === 0) continue;

         if (GSTREAMER_ERROR_PATTERN.test(line)) {
           const pipelineError: PipelineError = {
             code: "GSTREAMER_ERROR",
             message: `GStreamer error in pipeline "${this.config.label}"`,
             technicalDetails: line,
             timestamp: Date.now(),
           };
           this.emit("error", pipelineError);
         } else {
           logger.debug(`Pipeline "${this.config.label}" stderr: ${line}`, {
             pipelineId: this.id,
           });
         }
       }
     });
   }
   ```
