<objective>
Fix graceful shutdown requiring 3x Ctrl+C by adding a shutdown guard to PipelineManager and
cleaning up streaming event listeners.

Purpose: During graceful shutdown, the 5-second streaming drain period allows GStreamer pipelines
to crash (mediasoup transports close, UDP ports freed), triggering PipelineManager's auto-restart
scheduler which spawns new GStreamer processes. These orphaned processes require additional
Ctrl+C signals to kill. The fix adds an `isShuttingDown` flag that prevents restart scheduling,
and removes dangling event listeners from streaming to audio subsystem.

Output: Single Ctrl+C cleanly shuts down all subsystems with no orphaned processes.
</objective>

   To:
   ```
   // 1. Disable pipeline auto-restart BEFORE streaming teardown begins.
   //    Prevents orphaned GStreamer processes during 5s drain window.
   ```
