**1. Create `listener/src/hooks/usePreferences.ts`:**
localStorage-backed preference persistence:

```typescript
interface ListenerPreferences {
  lastChannelId: string | null;
  visitCount: number;
}
```

- `usePreferences()` hook returns:
  - `preferences: ListenerPreferences` — current values
  - `setLastChannel(channelId: string): void` — saves to localStorage
  - `incrementVisitCount(): void` — called once per app load (on mount)
  - `isReturningListener: boolean` — true if lastChannelId is set

- localStorage keys: `cas_last_channel`, `cas_visit_count`
- On mount: read from localStorage, parse, increment visit count
- `setLastChannel` called when audio starts playing on a channel
- NOTE: Volume does NOT persist (per locked decision — always starts at 70%)

**2. Create `listener/src/hooks/usePwaInstall.ts`:**
PWA install prompt on second visit (per locked decision):

- Captures the `beforeinstallprompt` event (Chrome/Edge fires this before showing native install banner)
- Stores the deferred prompt event
- Returns `{ canInstall: boolean, promptInstall: () => Promise<void> }`
- `canInstall` is true only when:
  1. The `beforeinstallprompt` event was captured AND
  2. `preferences.visitCount >= 2` (second visit per locked decision)
- `promptInstall()` calls `deferredPrompt.prompt()` and awaits the user choice
- On iOS Safari (no `beforeinstallprompt`): `canInstall` remains false. iOS users can still use "Add to Home Screen" manually.
- Clean up event listener on unmount

**3. Create `listener/src/hooks/useMediaSession.ts`:**
Media Session API for lock screen controls (per research code example):

```typescript
function useMediaSession(channelName: string, description: string, isPlaying: boolean, onPlay: () => void, onPause: () => void): void
```

- Sets `navigator.mediaSession.metadata` with:
  - `title`: channel name
  - `artist`: description (or "Church Audio Stream" if no description)
  - `album`: "Church Audio Stream"
- Sets action handlers:
  - "play" → calls `onPlay` callback (resumes audio engine)
  - "pause" → calls `onPause` callback (mutes audio engine)
- Updates `navigator.mediaSession.playbackState` based on `isPlaying` prop
- Guard: `if (!("mediaSession" in navigator)) return` — no-op on unsupported browsers
- Updates metadata when `channelName` or `description` changes
- Cleans up action handlers on unmount

**1. Create `listener/src/components/OfflineScreen.tsx`:**
Inline offline screen component (in addition to the static `public/offline.html` for SW fallback):
- Detects online/offline status via `navigator.onLine` + `online`/`offline` events
- Shows: "Connect to the church WiFi to listen to live translations" (per locked decision)
- WiFi icon (inline SVG), centered text, friendly tone
- "Try Again" button that checks `navigator.onLine` and reloads
- Full-screen overlay when offline
- Returns `null` when online (doesn't interfere with normal UI)

**2. Create `listener/src/components/ShareButton.tsx`:**
Share button using Web Share API with QR fallback (per locked decision):

Install qrcode package:
```bash
cd listener && npm install qrcode && npm install -D @types/qrcode
```

- Props: `listenerUrl: string` (the general listener URL, not channel-specific per locked decision)
- On tap:
  1. If `navigator.share` is available (most mobile browsers): call `navigator.share({ title: "Church Audio Stream", url: listenerUrl })`
  2. If not available (desktop, older browsers): show a modal with a QR code generated from the URL
- QR code generation: use `qrcode.toDataURL(url)` to generate a base64 PNG, display in an `<img>` tag
- Modal: centered overlay with QR code, URL text below, and a "Close" button
- The share button is a small icon button (share icon SVG) placed in the channel list header area
