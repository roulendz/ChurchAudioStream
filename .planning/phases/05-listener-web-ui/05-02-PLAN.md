**1. Create `listener/src/lib/signaling-client.ts` — protoo-client wrapper:**

Thin wrapper around protoo-client that manages WebSocket connection lifecycle:

```typescript
import * as protooClient from "protoo-client";
```

- `createSignalingPeer(wsUrl: string): protooClient.Peer` — creates a WebSocketTransport and Peer. The wsUrl should be `wss://{host}:{port}/ws/listener` (or `ws://` in dev).
- Export the Peer type for consumers.
- Do NOT add reconnection logic here — protoo-client has built-in retry (10 attempts, exponential backoff) per research. The caller (useSignaling hook) handles "open" and "disconnected" events.
- Helper `buildWsUrl(): string` that constructs the WebSocket URL from `window.location` — uses `wss://` for HTTPS, `ws://` for HTTP. Path is always `/ws/listener`.

**2. Create `listener/src/lib/mediasoup-device.ts` — mediasoup Device + transport factory:**

Manages the mediasoup-client Device singleton and receive transport creation:

```typescript
import * as mediasoupClient from "mediasoup-client";
```

- `loadDevice(rtpCapabilities: mediasoupClient.types.RtpCapabilities): Promise<mediasoupClient.types.Device>` — creates and loads a Device with router capabilities. Caches the device instance (only load once per session, unless reconnecting).
- `createRecvTransport(device: mediasoupClient.types.Device, transportInfo: TransportInfo): mediasoupClient.types.Transport` — creates a receive transport from server-provided params (id, iceParameters, iceCandidates, dtlsParameters). Returns the transport without wiring the "connect" event — the caller wires it with the signaling peer.
- Type `TransportInfo = { id: string; iceParameters: object; iceCandidates: object[]; dtlsParameters: object }` for the server response shape.
- `resetDevice(): void` — clears the cached device (used on reconnection after WiFi drop).

**3. Create `listener/src/lib/audio-engine.ts` — Web Audio API engine:**

Per research Pattern 2 (iOS Safari volume fix). All audio routing goes through GainNode:

```typescript
interface AudioEngine {
  playTrack(track: MediaStreamTrack): void;
  setVolume(value: number): void;  // 0.0 to 1.0
  mute(): void;
  unmute(volume: number): void;
  isMuted(): boolean;
  resume(): Promise<void>;  // Must be called from user gesture
  getContext(): AudioContext;
  close(): void;
}
```

- `createAudioEngine(): AudioEngine`
- Internal: creates `AudioContext` (NOT resumed yet — that happens on "Start Listening" tap), `GainNode` at 0.7 default (70% per locked decision), connected to `audioContext.destination`.
- `playTrack`: creates `MediaStream([track])`, creates `MediaStreamSourceNode`, connects to GainNode. Stores reference so it can be disconnected later. If a previous source exists, disconnect it first (channel switch case).
- `setVolume`: uses `gainNode.gain.setValueAtTime(value, audioContext.currentTime)` for glitch-free updates.
- `mute/unmute`: stores the current volume, sets gain to 0 / restores. Tracks muted state internally.
- `resume`: calls `audioContext.resume()` — MUST be called inside a user gesture handler.
- `close`: calls `audioContext.close()`, nullifies references.
- Handle `visibilitychange` event: when document becomes visible again, call `audioContext.resume()` if it was suspended (mobile browser power saving per research pitfall 5).

**CRITICAL:** Do NOT use `new Audio()` or `HTMLAudioElement`. All audio goes through Web Audio API per research finding #1 (iOS Safari volume is read-only on HTMLAudioElement).

**1. Create `listener/src/hooks/useSignaling.ts`:**
Custom hook managing protoo peer lifecycle:
- Creates peer via `createSignalingPeer(buildWsUrl())`
- State: `connectionState: "connecting" | "connected" | "disconnected" | "reconnecting"`
- Listens to peer events: "open" (connected), "close" / "disconnected" (handle reconnection state)
- Exposes `peer` object for making requests, `connectionState` for UI
- On "open" event after a reconnection (not first connect), sets `isReconnect` flag for the mediasoup hook to re-run full handshake (per research pitfall 5)
- Cleanup: close peer on unmount

**2. Create `listener/src/hooks/useChannelList.ts`:**
Custom hook managing the channel list from server notifications:
- Listens to peer notification "activeChannels" → updates channel list state + defaultChannelId
- Listens to peer notification "listenerCounts" → merges updated counts into existing channel list
- Listens to peer notification "channelStopped" → updates affected channel to offline
- Returns `{ channels: ListenerChannelInfo[], defaultChannelId: string | null }`
- Sorts channels: live (hasActiveProducer=true) first, offline last. Within each group, preserve server order (admin-defined).
- Type `ListenerChannelInfo` matches the server-side type (define locally or import from a shared types file)

**3. Create `listener/src/hooks/useMediasoup.ts`:**
Custom hook managing mediasoup Device, transport, and consumer:
- `connectToChannel(channelId: string, peer: protooClient.Peer): Promise<MediaStreamTrack>` — runs the full signaling handshake:
  1. `peer.request("getRouterRtpCapabilities")` → `loadDevice(rtpCapabilities)`
  2. `peer.request("createWebRtcTransport", { rtpCapabilities: device.rtpCapabilities, channelId })` → `createRecvTransport(device, transportInfo)`
  3. Wire transport "connect" event → `peer.request("connectWebRtcTransport", { dtlsParameters })`
  4. `peer.request("consume", { channelId })` → `transport.consume({ id, producerId, kind, rtpParameters })`
  5. `peer.request("resumeConsumer")` → return `consumer.track`
- `disconnect(): void` — closes transport and consumer (clean break per locked decision)
- On peer reconnection (isReconnect flag from useSignaling), call `resetDevice()` and re-run handshake if a channel was active

**4. Create `listener/src/hooks/useAudioPlayback.ts`:**
Custom hook wrapping the audio engine:
- Creates engine via `createAudioEngine()` on mount
- `startPlayback(track: MediaStreamTrack): Promise<void>` — calls `engine.resume()` then `engine.playTrack(track)`
- `setVolume(value: number)`, `mute()`, `unmute()`, `isMuted` — delegates to engine
- `stopPlayback()` — disconnects current source
- Cleanup: close engine on unmount
