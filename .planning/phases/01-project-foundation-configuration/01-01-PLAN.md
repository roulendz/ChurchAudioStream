<task type="auto">
  <name>Task 3: Implement Tauri Rust sidecar lifecycle management</name>
  <files>
    src-tauri/src/lib.rs
    src-tauri/Cargo.toml
  </files>
  <action>
    1. Update `src-tauri/Cargo.toml`:
       - Ensure tauri-plugin-shell is in dependencies
       - Ensure tokio features include "time" (for restart delay)

    2. Implement sidecar lifecycle in `src-tauri/src/lib.rs`:

       a. Register the shell plugin in the Tauri builder:
          `.plugin(tauri_plugin_shell::init())`

       b. Create a `spawn_sidecar` function (following research Pattern 1):
          - Use `app.shell().sidecar("server")` to create sidecar command
          - Pass `--config-path` argument pointing to config.json location next to the executable
          - Spawn the sidecar and get (rx, child) channels
          - Loop over CommandEvent variants:
            - Stdout: emit "sidecar-log" event to frontend with the line content
            - Stderr: emit "sidecar-error" event to frontend
            - Error: emit "sidecar-crash" event, break to trigger restart
            - Terminated: emit "sidecar-crash" with exit code info, break to trigger restart
          - After loop exit (crash/terminate), if should_run flag is true, sleep 2 seconds then re-enter spawn loop (auto-restart)
          - Use an Arc<AtomicBool> `should_run` flag to control the restart loop

       c. Handle app close for clean shutdown:
          - Use `.on_window_event(|window, event|)` to listen for CloseRequested
          - Set should_run to false
          - Kill the sidecar child process explicitly
          - This prevents orphaned sidecar processes on Windows (Pitfall 2 from research)

       d. Start the sidecar in the `setup` hook:
          - In `.setup(|app| { ... })`, call spawn_sidecar
          - Store the should_run Arc in app state for access from window event handler

    IMPORTANT: Do NOT use Tauri IPC (#[tauri::command]) for admin operations. The admin GUI connects via WebSocket (research decision: "admin GUI is just another client"). Tauri Rust side ONLY manages sidecar lifecycle.
    IMPORTANT: The sidecar binary name in shell().sidecar("server") must match the prefix in externalBin config ("binaries/server"). Tauri automatically appends the target triple suffix.
  </action>

    - `src-tauri/src/lib.rs` handles CloseRequested event to kill sidecar
    - `src-tauri/src/lib.rs` registers tauri_plugin_shell
    - Build the sidecar first (`cd sidecar && npm run build`), then run `npm run tauri dev` -- the Tauri window should open and sidecar logs should appear in the terminal
