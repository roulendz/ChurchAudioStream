  <files>

  <action>
    **Create `src/hooks/useChannels.ts`:**
    - Accept `sendMessage` and `subscribe` from useServerStatus (same function signatures).
    - State: `channels: AppChannel[]` (use a local `AdminChannel` type mirroring the server AppChannel interface with all fields needed by admin: id, name, sources, outputFormat, autoStart, status, processing, visible, sortOrder, createdAt).
    - On mount:
      - Send `channels:list` to request initial data.
      - Subscribe to `channels:list`, `channel:created`, `channel:updated`, `channel:removed`, `channel:state`.
    - `channels:list` handler: replace full list via `setChannels(payload.channels)`.
    - `channel:created` handler: append to list, re-sort by sortOrder.
    - `channel:updated` handler: replace matching channel by id, re-sort.
    - `channel:removed` handler: filter out by channelId.
    - `channel:state` handler: update status field on matching channel.
    - Expose CRUD operation functions (these send WS messages, optimistic update is NOT needed since server broadcasts back):
      - `createChannel(name: string, outputFormat?: "mono" | "stereo")` -> sends `channel:create`.
      - `updateChannel(channelId: string, updates: { name?: string; outputFormat?: string; autoStart?: boolean; visible?: boolean })` -> sends `channel:update`.
      - `removeChannel(channelId: string)` -> sends `channel:remove`.
      - `startChannel(channelId: string)` -> sends `channel:start`.
      - `stopChannel(channelId: string)` -> sends `channel:stop`.
      - `reorderChannels(channelIds: string[])` -> sends `channel:reorder`.
      - `addSource(channelId: string, sourceId: string, selectedChannels: number[])` -> sends `channel:source:add`.
      - `removeSource(channelId: string, sourceIndex: number)` -> sends `channel:source:remove`.
    - Cleanup: return all unsubscribe functions from useEffect.
    - Return `{ channels, createChannel, updateChannel, removeChannel, startChannel, stopChannel, reorderChannels, addSource, removeSource }`.

    **Create `src/hooks/useSources.ts`:**
    - Accept `sendMessage` and `subscribe`.
    - State: `sources: DiscoveredSource[]` (local type mirroring the server DiscoveredSource discriminated union: id, type, name, status, channelCount, channelLabels for AES67; id, type, name, status, channelCount, api, isLoopback for local).
    - On mount: send `sources:list`, subscribe to `sources:list` and `sources:changed`.
    - `sources:list` response: replace full list.
    - `sources:changed` notification: re-request via `sendMessage("sources:list")`.
    - Cleanup: return unsubscribes.
    - Return `{ sources }`.

    **Type definitions:** Define local TypeScript interfaces in each hook file (NOT importing from sidecar). The admin types mirror the server-side shapes but are independent (same pattern as listener types in listener/src/lib/types.ts).

    **Important:** Use `useCallback` for the operation functions to maintain stable references. Wrap subscriptions in a single `useEffect` with all unsubscribes in the cleanup return. Request initial data only when `connectionStatus === "connected"` if that signal is available, OR simply send on mount (the WS hook buffers messages and identify is sent first).
  </action>
