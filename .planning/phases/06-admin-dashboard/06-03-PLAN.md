  <action>
    **Create `src/hooks/useAudioLevels.ts`:**
    - Accept `subscribe` function from useServerStatus.
    - Store level data in a `useRef<Map<string, ChannelLevelData>>` where key is channelId.
    - Define local type `ChannelLevelData`:
      ```typescript
      interface ChannelLevelData {
        peak: number[];      // 0.0-1.0 per audio channel
        rms: number[];       // 0.0-1.0 per audio channel
        clipping: boolean;
        gainReductionDb: number;
        timestamp: number;
      }
      ```
    - Subscribe to `levels:update`. The payload is `{ levels: Record<pipelineId, LevelEntry> }` where each LevelEntry now has a `channelId` field (added by Plan 01).
    - On each levels:update, iterate entries and group by channelId:
      - For each channelId, take the max peak and max rms across all pipelines belonging to that channel.
      - Merge clipping (OR across pipelines).
      - Store the merged result in the ref map.
    - Expose a `getLevels(channelId: string): ChannelLevelData | null` callback that reads from the ref (does NOT trigger re-render).
    - Expose a `getActiveLevelChannelIds(): string[]` callback that returns channelIds with recent data (timestamp within last 1 second).
    - Return `{ getLevels, getActiveLevelChannelIds }`.

    **CRITICAL:** This hook MUST NOT use useState for level data. All data stays in useRef. The Canvas rAF loop reads from the ref directly. This prevents 10x/sec re-renders.

    **Create `src/components/monitoring/VuMeter.tsx`:**
    - Accept: `channelId: string`, `channelName: string`, `getLevels: (id: string) => ChannelLevelData | null`, `width?: number` (default 32), `height?: number` (default 200).
    - Use `useRef<HTMLCanvasElement>` for the canvas.
    - Handle HiDPI: scale canvas by `window.devicePixelRatio`, use CSS for display size (per research pitfall #3).
    - Run a `requestAnimationFrame` loop in a `useEffect`:
      - Read `getLevels(channelId)` each frame.
      - If null, draw empty/grey meter.
      - Otherwise, draw:
        - Background: dark grey (var --bg-primary equivalent: #1a1a2e).
        - RMS bar: semi-transparent accent color, height = rms[0] * canvasHeight (use max of channels).
        - Peak bar: brighter accent color, narrow line at peak level.
        - Clipping indicator: small rectangle at top, red (#f44336) when clipping, dark grey otherwise.
        - Gain reduction: optional small indicator showing AGC activity.
      - Use smooth decay: if new level < current drawn level, decay by ~0.92 factor per frame (avoids jumpy meters). Store smoothed values in a separate ref.
    - Below the canvas, display the channel name in a small label.
    - Cleanup: cancel rAF on unmount.

    **Create `src/components/monitoring/VuMeterBank.tsx`:**
    - Accept: `channels` array (from useChannels), `getLevels` function (from useAudioLevels).
    - Render a horizontal flex/grid row of `VuMeter` components, one per channel.
    - Filter to only show channels with status "streaming" or "starting" (active channels with potential audio).
    - If no active channels, show a muted message: "No active channels -- start a channel to see audio levels."
    - Style: flex row with gap, centered, wrapping if many channels.
