  <action>
    **Gap 1 -- Channel visibility field:**
    - In `channel-types.ts`, add `visible: boolean` to the `AppChannel` interface (mutable, not readonly).
    - In `schema.ts`, add `visible: z.boolean().default(true)` to `ChannelSchema`.
    - In `channel-manager.ts`:
      - Add `visible` to the `ChannelUpdatableFields` type (extend the Pick).
      - In `createChannel()`, set `visible: true` in the new channel object.
      - In `updateChannel()`, accept and apply `visible` field updates.
      - In `loadChannelsFromConfig()`, ensure existing channels without `visible` get `true` default (Zod default handles this).

    **Gap 2 -- Channel sortOrder field and reorder API:**
    - In `channel-types.ts`, add `sortOrder: number` to `AppChannel` (mutable).
    - In `schema.ts`, add `sortOrder: z.number().int().min(0).default(0)` to `ChannelSchema`.
    - In `channel-manager.ts`:
      - In `createChannel()`, set `sortOrder` to `this.channels.size` (appends at end).
      - Add method `reorderChannels(orderedIds: string[]): AppChannel[]` that:
        - Validates all IDs exist and array length matches channels count.
        - Updates `sortOrder` on each channel to match the new array index.
        - Calls `persistChannels()`.
        - Emits "channel-updated" for each changed channel.
        - Returns the reordered channel list.
      - Update `getAllChannels()` to return channels sorted by `sortOrder` (ascending).
    - In `audio-subsystem.ts`, add `reorderChannels(orderedIds: string[]): AppChannel[]` passthrough to channelManager.
    - In `types.ts`, add `ChannelReorderPayload` interface: `{ channelIds: string[] }`.
    - In `handler.ts`, add `"channel:reorder"` case in `handleAudioMessageAsync`:
      - Validate `channelIds` is a non-empty string array.
      - Call `audioSubsystem.reorderChannels(payload.channelIds)`.
      - Send `"channels:list"` response with `{ channels: reorderedList }`.
    - Also in `handler.ts`, update the `ChannelUpdatePayload` import/type to include `visible?: boolean`.
    - In the `"channel:update"` handler, pass `visible: payload.visible` through to `audioSubsystem.updateChannel()`.

    **Gap 3 -- channelId in level broadcast:**
    - In `handler.ts` function `wireAudioBroadcasts`, the level event handler receives `levels: { pipelineId: string }`. This data is stored in `levelBuffer` keyed by `pipelineId`.
    - The admin UI needs to know which channel each pipelineId belongs to.
    - Add a `getPipelineToChannelMap(): Map<string, string>` method to `ChannelManager` that iterates `channelPipelines` and returns a reverse map of `pipelineId -> channelId`.
    - Expose this through `AudioSubsystem` as `getPipelineToChannelMap()`.
    - In `wireAudioBroadcasts`, when flushing the level buffer, enrich each level entry with a `channelId` field using the pipeline-to-channel map:
      ```
      const pipelineToChannel = audioSubsystem.getPipelineToChannelMap();
      for (const [pipelineId, levelData] of Object.entries(levelBuffer)) {
        (levelData as any).channelId = pipelineToChannel.get(pipelineId) ?? null;
      }
      ```
    - This is LOW overhead (map lookup on flush, not per-frame).

    **Important:** Ensure TypeScript compiles (`cd sidecar && npx tsc --noEmit`). Do NOT change the `ChannelUpdatePayload` in `types.ts` -- extend it to include `visible?: boolean`.
  </action>
