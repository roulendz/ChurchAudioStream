  <name>Task 1: Create GStreamer pipeline string builder</name>
  <files>sidecar/src/audio/pipeline/pipeline-builder.ts</files>
  <action>
    Create `pipeline-builder.ts` that exports a single entry point `buildPipelineString(config: PipelineConfig): string` which delegates to internal builder functions based on source type.

    Import `PipelineConfig` and `AudioApi` from `./pipeline-types` (will exist from Plan 01; if building in parallel, define a minimal local type and note it for reconciliation).

    **Internal builder functions (not exported, SRP):**

    1. `buildAes67Pipeline(aes67Config, levelIntervalNs)` -- AES67 multicast RTP receive:
       - `udpsrc address={multicastAddress} port={port} caps="application/x-rtp, clock-rate={sampleRate}, channels={channelCount}, payload={payloadType}" buffer-size=65536`
       - `! rtpjitterbuffer latency=5`
       - `! rtpL{bitDepth}depay` -- use bitDepth to select rtpL16depay or rtpL24depay (from RESEARCH open question #5: parse SDP to determine, don't rely on auto-negotiation)
       - If selectedChannels.length === 1 AND channelCount > 1: add `! deinterleave name=d d.src_{selectedChannels[0]} ! queue`
       - If selectedChannels.length === 2 AND channelCount > 2: add deinterleave + interleave pattern: `! deinterleave name=d d.src_{ch[0]} ! queue ! interleave name=i d.src_{ch[1]} ! queue ! i. i.`
       - If selectedChannels.length === channelCount: no deinterleave needed (take all channels)
       - End with `! audioconvert ! audioresample ! level interval={levelIntervalNs} post-messages=true ! fakesink sync=false`

    2. `buildWasapiPipeline(localConfig, levelIntervalNs)` -- WASAPI capture:
       - `wasapi2src device="{deviceId}" low-latency=true`
       - If isLoopback: `wasapi2src loopback=true` (no device param for default loopback, or with device if specified)
       - Channel selection: if selectedChannels doesn't span all device channels, add deinterleave like AES67
       - End with `! audioconvert ! audioresample ! level interval={levelIntervalNs} post-messages=true ! fakesink sync=false`

    3. `buildAsioPipeline(localConfig, levelIntervalNs)` -- ASIO capture:
       - `asiosrc device-clsid="{deviceId}" input-channels="{selectedChannels.join(',')}"` -- ASIO element supports native channel selection
       - If bufferSize specified: `buffer-size={bufferSize}`
       - End with `! audioconvert ! audioresample ! level interval={levelIntervalNs} post-messages=true ! fakesink sync=false`

    4. `buildDirectSoundPipeline(localConfig, levelIntervalNs)` -- DirectSound capture (fallback):
       - `directsoundsrc device-name="{deviceId}"`
       - Channel selection via deinterleave if needed
       - End with `! audioconvert ! audioresample ! level interval={levelIntervalNs} post-messages=true ! fakesink sync=false`

    **Helper function:**
    - `msToGstNanoseconds(ms: number): number` -- converts levelIntervalMs to GStreamer nanoseconds (multiply by 1_000_000). GStreamer level element `interval` property is in nanoseconds.

    **The main `buildPipelineString` function:**
    - Validate that exactly one of aes67Config or localConfig is provided (throw descriptive error otherwise)
    - Convert levelIntervalMs to nanoseconds
    - Delegate to the appropriate builder
    - Return the complete pipeline string

    **Important:** Properly escape device IDs in pipeline strings. Windows device paths contain backslashes and curly braces. Use double-quoting in the pipeline string for device identifiers. For ASIO CLSIDs like `{xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx}`, ensure the curly braces are inside quotes.

    **Anti-pattern to avoid:** Do NOT hard-code pipeline strings. Each builder function constructs the string from config parameters.

<task type="auto">
  <name>Task 2: Create GStreamer level metering stderr parser</name>
  <files>sidecar/src/audio/pipeline/metering-parser.ts</files>
  <action>
    Create `metering-parser.ts` that parses GStreamer level element messages from `gst-launch-1.0 -m` stderr output.

    Import `AudioLevels` from `./pipeline-types`.

    **Exported functions:**

    1. `parseMeteringLine(line: string): AudioLevels | null`
       - Match against the GStreamer level message format from stderr when using `-m` flag
       - Level messages look like: `level, endtime=(guint64)100000000, timestamp=(guint64)0, stream-time=(guint64)0, running-time=(guint64)100000000, duration=(guint64)100000000; peak=(double){ -12.5, -14.2 }, rms=(double){ -18.3, -20.1 }, decay=(double){ -13.0, -15.0 };`
       - Note: The format varies. Single-channel streams omit the curly braces: `peak=(double)-12.5, rms=(double)-18.3, decay=(double)-13.0`
       - Use a regex that handles BOTH formats: with and without curly braces, single and multi-channel
       - Extract peak, rms, and decay arrays. If decay is not found, return empty array.
       - Set `clipping: true` if any peak value >= -0.1
       - Set `timestamp: Date.now()`
       - Return null for non-level lines (debug messages, errors, state changes, etc.)

    2. `dbToNormalized(db: number): number`
       - Convert dB to 0.0-1.0 range for display: `Math.max(0, Math.min(1, Math.pow(10, db / 20)))`
       - -60 dB maps to ~0.001 (essentially 0), 0 dB maps to 1.0
       - Negative infinity (GStreamer sends `-inf` for silence) should map to 0

    3. `createStderrLineParser(onLevels: (levels: AudioLevels) => void, onError: (line: string) => void): (chunk: Buffer) => void`
       - Returns a function that processes raw stderr Buffer chunks
       - Splits on newlines, handling partial lines across chunks (accumulate buffer between newline boundaries)
       - For each complete line: if parseMeteringLine returns non-null, call onLevels; if line contains "ERROR" or "WARN" from GStreamer, call onError
       - This handles the stderr buffer overflow pitfall (Pitfall 5 from RESEARCH): process lines immediately as they arrive, never accumulate unbounded data

    **Regex patterns:**
    - Level message identifier: `/^.*level,/` (lines containing "level," are level messages)
    - Peak extraction: `/peak=\(double\)\{?\s*([-\d.e+inf]+(?:,\s*[-\d.e+inf]+)*)\s*\}?/`
    - RMS extraction: `/rms=\(double\)\{?\s*([-\d.e+inf]+(?:,\s*[-\d.e+inf]+)*)\s*\}?/`
    - Decay extraction: `/decay=\(double\)\{?\s*([-\d.e+inf]+(?:,\s*[-\d.e+inf]+)*)\s*\}?/`
    - Handle `-inf` as `Number.NEGATIVE_INFINITY` in parseFloat (it naturally parses to -Infinity)

    **Important:** The parser must be robust. GStreamer stderr can contain intermixed debug output, state change notifications, and error messages alongside level data. Only extract from lines that match the level message pattern.
  </action>
  <verify>

    4. Non-level line: `parseMeteringLine('Setting pipeline to PAUSED ...')` returns null

    6. dbToNormalized(0) === 1.0, dbToNormalized(-60) is near 0, dbToNormalized(-Infinity) === 0
  </verify>
