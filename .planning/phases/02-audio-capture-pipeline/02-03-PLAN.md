  <name>Task 1: Create GStreamerProcess child process wrapper</name>
  <files>sidecar/src/audio/pipeline/gstreamer-process.ts</files>
  <action>
    Create `gstreamer-process.ts` that exports a `GStreamerProcess` class wrapping one gst-launch-1.0 child process.

    **Imports:**
    - `spawn, ChildProcess` from `node:child_process`
    - `EventEmitter` from `node:events`
    - `buildPipelineString` from `./pipeline-builder`
    - `createStderrLineParser` from `./metering-parser`
    - `PipelineConfig, PipelineState, AudioLevels, PipelineError, PipelineStats` from `./pipeline-types`
    - `logger` from `../../utils/logger`

    **GStreamerProcess extends EventEmitter with typed events:**
    - `"state-change"` -> (state: PipelineState)
    - `"levels"` -> (levels: AudioLevels)
    - `"error"` -> (error: PipelineError)
    - `"exit"` -> (code: number | null, signal: string | null)

    **Constructor:** `constructor(config: PipelineConfig)`
    - Store config
    - Initialize state to "stopped"
    - Do NOT start the process in the constructor

    **Properties:**
    - `readonly id: string` (unique ID for this pipeline instance, use crypto.randomUUID())
    - `state: PipelineState` (current lifecycle state, starts "stopped")
    - `pid: number | null` (child process PID when running)
    - `startedAt: number | null` (timestamp when last started)
    - `config: PipelineConfig` (readonly)

    **Method: `start(): void`**
    - If already running (state is "initializing", "connecting", "buffering", or "streaming"), throw error
    - Set state to "initializing", emit "state-change"
    - Build pipeline string via `buildPipelineString(this.config)`
    - Spawn: `spawn('gst-launch-1.0', ['-m', '-e', ...pipelineString.split(' ')])` -- WAIT, gst-launch-1.0 needs the ENTIRE pipeline as separate args or as a single string. The correct approach is to pass the pipeline elements as part of the args array. Actually, `gst-launch-1.0` accepts the pipeline description as positional arguments. Split on whitespace but be careful with quoted strings. A simpler approach: use `spawn('gst-launch-1.0', ['-m', '-e', pipelineString], { shell: true })` with shell: true to handle the pipeline string properly. This avoids complex argument splitting.
    - Set `stdio: ['pipe', 'pipe', 'pipe']`
    - Set `env: { ...process.env }` to inherit GStreamer PATH
    - Store the ChildProcess reference and PID
    - Attach stderr parser via `createStderrLineParser`:
      - onLevels: emit "levels" event, update state to "streaming" if still "initializing"/"connecting"/"buffering" (first level data = pipeline is working)
      - onError: emit "error" event with parsed PipelineError
    - Attach stdout data handler (log debug output)
    - Attach "exit" handler:
      - If exit code 0 or we requested stop: set state "stopped"
      - If non-zero exit code and we did NOT request stop: set state "crashed", emit "error" with PROCESS_CRASH code
      - Emit "exit" event
      - Clean up references (set process/pid to null)
    - Attach "error" handler on the ChildProcess (spawn failure):
      - Set state "crashed", emit "error" with descriptive message (e.g., "gst-launch-1.0 not found" if ENOENT)
    - Set state to "connecting" after successful spawn, emit "state-change"
    - Record startedAt timestamp

    **Method: `stop(): Promise<void>`**
    - If not running, return immediately
    - Set state to "stopping", emit "state-change"
    - Send SIGTERM to the child process (gst-launch-1.0 with -e flag handles EOS cleanly on SIGTERM)
    - Wait for exit with timeout (drainTimeoutMs from config, default 500ms)
    - If process doesn't exit within timeout, send SIGKILL
    - Return a promise that resolves when the process has exited

    **Method: `getUptime(): number`**
    - Returns Date.now() - startedAt if running, otherwise 0

    **Important implementation notes:**
    - ALWAYS use `shell: true` on Windows for gst-launch-1.0 spawning -- the pipeline string contains characters (!, =, quotes) that need shell interpretation
    - The `-e` flag tells gst-launch-1.0 to handle EOS on interrupt (SIGTERM/Ctrl+C), enabling graceful shutdown
    - The `-m` flag enables bus message output on stderr (required for level metering)
    - On state transitions, log with the pipeline label from config for easy debugging
    - Handle the ENOENT spawn error specifically: if gst-launch-1.0 is not found, provide clear error "GStreamer not found. Install GStreamer 1.26 and add to PATH."
    - Per RESEARCH Pitfall 3: NEVER reuse a gst-launch-1.0 process. Always kill and spawn new.
  </action>

  <done>
    GStreamerProcess wraps a single gst-launch-1.0 child process with full lifecycle management. State transitions follow the documented lifecycle (initializing -> connecting -> buffering -> streaming | stopped | crashed). stderr is parsed for level data and errors. Graceful shutdown with SIGTERM + SIGKILL fallback. Spawn errors (gst-launch not found) produce clear error messages.

  <name>Task 2: Create PipelineManager registry with crash recovery</name>
  <files>sidecar/src/audio/pipeline/pipeline-manager.ts</files>
  <action>
    Create `pipeline-manager.ts` that exports a `PipelineManager` class -- the registry and coordinator for all active GStreamer pipelines.

    **Imports:**
    - `GStreamerProcess` from `./gstreamer-process`
    - `PipelineConfig, PipelineState, AudioLevels, PipelineError, PipelineStats` from `./pipeline-types`
    - `EventEmitter` from `node:events`
    - `logger` from `../../utils/logger`

    **PipelineManager extends EventEmitter with events:**
    - `"pipeline-state-change"` -> (pipelineId: string, state: PipelineState)
    - `"pipeline-levels"` -> (pipelineId: string, levels: AudioLevels)
    - `"pipeline-error"` -> (pipelineId: string, error: PipelineError)

    **Internal state:**
    - `pipelines: Map<string, GStreamerProcess>` -- all managed pipelines keyed by pipeline ID
    - `restartAttempts: Map<string, number>` -- crash restart counter per pipeline ID
    - `restartTimers: Map<string, NodeJS.Timeout>` -- pending restart timers for cleanup
    - `recoveryConfig: { autoRestart, maxRestartAttempts, restartDelayMs, drainTimeoutMs }` -- from config

    **Constructor:** `constructor(recoveryConfig: { autoRestart: boolean, maxRestartAttempts: number, restartDelayMs: number, drainTimeoutMs: number })`

    **Method: `createPipeline(config: PipelineConfig): string`**
    - Create new GStreamerProcess(config)
    - Wire up event listeners:
      - On "state-change": re-emit as "pipeline-state-change" with pipeline ID. If state is "streaming", reset restart attempts for this pipeline to 0.
      - On "levels": re-emit as "pipeline-levels" with pipeline ID
      - On "error": re-emit as "pipeline-error" with pipeline ID
      - On "exit": if state is "crashed" and autoRestart is enabled and attempts < max, schedule restart after delay. Log the restart attempt.
    - Add to pipelines map
    - Return pipeline ID

    **Method: `startPipeline(pipelineId: string): void`**
    - Get pipeline from map, throw if not found
    - Call pipeline.start()

    **Method: `stopPipeline(pipelineId: string): Promise<void>`**
    - Get pipeline from map, throw if not found
    - Clear any pending restart timer
    - Call pipeline.stop()

    **Method: `removePipeline(pipelineId: string): Promise<void>`**
    - Stop the pipeline if running
    - Remove all event listeners from the GStreamerProcess
    - Delete from all maps (pipelines, restartAttempts, restartTimers)

    **Method: `getPipelineState(pipelineId: string): PipelineState | null`**
    - Return current state of pipeline, or null if not found

    **Method: `getAllPipelineIds(): string[]`**
    - Return array of all pipeline IDs

    **Method: `getPipelineConfig(pipelineId: string): PipelineConfig | null`**
    - Return config of pipeline, or null if not found

    **Method: `stopAll(): Promise<void>`**
    - Stop all pipelines concurrently (Promise.allSettled)
    - Clear all restart timers
    - Log summary of stopped pipelines

    **Method: `destroyAll(): Promise<void>`**
    - Remove all pipelines (stop + cleanup)
    - Clear all maps

    **Crash recovery logic (private):**
    - `scheduleRestart(pipelineId: string)`:
      - Increment restartAttempts for this pipeline
      - If attempts > maxRestartAttempts: log "Max restart attempts reached for {label}", emit error, do NOT restart
      - Otherwise: set timeout for restartDelayMs, then call pipeline.start()
      - Store the timeout handle in restartTimers for cleanup

    **Important:**
    - Pipeline IDs are the GStreamerProcess.id (UUID), NOT the channel ID. One channel may have multiple pipelines (multiple source assignments). The channel manager (Plan 08) maps channel IDs to pipeline IDs.
    - restartAttempts reset to 0 when a pipeline reaches "streaming" state (successful recovery resets the counter)
    - On removePipeline, always clear restart timer to avoid zombie restarts
  </action>
  <verify>
    `npx tsc --noEmit` passes. PipelineManager class exports with all documented methods. Verify event forwarding pattern: GStreamerProcess events -> PipelineManager events with pipeline ID prefix. Verify crash recovery logic: restart attempts increment, max attempts stop recovery, successful streaming resets counter.
  </verify>

Events are forwarded with pipeline IDs for downstream consumers (WebSocket broadcast). stopAll and destroyAll handle graceful multi-pipeline shutdown.
