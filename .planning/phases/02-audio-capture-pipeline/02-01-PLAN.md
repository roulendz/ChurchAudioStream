  <name>Task 1: Install npm dependencies and create audio type definitions</name>
  <files>
    sidecar/package.json

  <action>
    **Step 1: Install dependencies**
    Run `npm install sdp-transform pidusage` and `npm install -D @types/sdp-transform` in the sidecar directory.

    **Step 2: Create source-types.ts**
    Define types for all audio source kinds:

    - `AudioApi` = `"wasapi2"` | `"asio"` | `"directsound"` -- the three Windows audio APIs
    - `SourceStatus` = `"available"` | `"unavailable"` | `"in-use"` | `"verifying"`
    - `AES67Source` -- represents a discovered AES67/Dante multicast stream:
      - `id: string` (unique, derived from multicast address + port)
      - `type: "aes67"` (discriminator)
      - `name: string` (from SDP session name `s=` field)
      - `description: string` (from SDP `i=` field, empty string if absent)
      - `multicastAddress: string`
      - `port: number`
      - `sampleRate: number` (e.g., 48000)
      - `bitDepth: number` (16 or 24, from SDP rtpmap L16/L24)
      - `channelCount: number` (total channels in stream)
      - `payloadType: number` (RTP payload type from SDP)
      - `originAddress: string` (SAP origin IP)
      - `channelLabels: string[]` (from SDP `a=label:` attributes, empty array if none; length should match channelCount)
      - `status: SourceStatus`
      - `lastSeenAt: number` (Unix timestamp ms of last SAP announcement)
      - `discoveredAt: number` (Unix timestamp ms)
    - `LocalDeviceSource` -- represents a local audio input device:
      - `id: string` (unique, derived from deviceId + api)
      - `type: "local"` (discriminator)
      - `name: string` (device display name from GStreamer)
      - `api: AudioApi`
      - `deviceId: string` (wasapi2 device path, ASIO CLSID, DirectSound device name)
      - `sampleRate: number`
      - `bitDepth: number`
      - `channelCount: number`
      - `isLoopback: boolean` (true for WASAPI loopback capture)
      - `status: SourceStatus`
      - `lastSeenAt: number`
    - `DiscoveredSource` = `AES67Source | LocalDeviceSource` (union type)

    **Step 3: Create channel-types.ts**
    Define types for app channels:

    - `ChannelOutputFormat` = `"mono"` | `"stereo"`
    - `ChannelStatus` = `"stopped"` | `"starting"` | `"streaming"` | `"error"` | `"crashed"`
    - `SourceAssignment` -- a source assigned to an app channel:
      - `sourceId: string` (references DiscoveredSource.id)
      - `selectedChannels: number[]` (0-based indices, e.g., [0] for mono, [0,1] for stereo pair)
      - `gain: number` (0.0 to 2.0, default 1.0 -- linear gain multiplier)
      - `muted: boolean` (default false)
      - `delayMs: number` (default 0, offset in ms for time-alignment)
    - `AppChannel` -- an app-level audio channel:
      - `id: string` (UUID)
      - `name: string` (display name, admin-editable in Phase 6)
      - `sources: SourceAssignment[]` (multiple sources, mixing in Phase 3)
      - `outputFormat: ChannelOutputFormat` (default "mono")
      - `autoStart: boolean` (default true -- start pipeline on app launch)
      - `status: ChannelStatus`
      - `createdAt: number` (Unix timestamp ms)

    **Step 4: Create pipeline-types.ts**
    Define types for pipeline process management:

    - `PipelineState` = `"initializing"` | `"connecting"` | `"buffering"` | `"streaming"` | `"stopping"` | `"stopped"` | `"crashed"`
    - `AudioLevels` -- parsed from GStreamer level element:
      - `peak: number[]` (dB per channel)
      - `rms: number[]` (dB per channel)
      - `decay: number[]` (dB per channel)
      - `timestamp: number` (Date.now() when parsed)
      - `clipping: boolean` (true if any peak >= -0.1 dB)
    - `PipelineStats` -- runtime stats:
      - `cpuPercent: number`
      - `memoryMb: number`
      - `uptimeMs: number`
      - `pid: number`
    - `PipelineError` -- structured pipeline error:
      - `code: string` (e.g., "DEVICE_BUSY", "NO_MULTICAST", "PROCESS_CRASH", "PIPELINE_ERROR")
      - `message: string` (user-friendly)
      - `technicalDetails: string` (full GStreamer error)
      - `timestamp: number`
    - `PipelineConfig` -- config needed to construct and run a pipeline:
      - `sourceType: "aes67" | "local"`
      - `aes67Config?: { multicastAddress, port, sampleRate, channelCount, bitDepth, payloadType, selectedChannels: number[] }`
      - `localConfig?: { deviceId, api: AudioApi, selectedChannels: number[], bufferSize?: number, isLoopback?: boolean }`
      - `levelIntervalMs: number` (default 100, configurable: 100 or 33)
      - `label: string` (for logging, e.g., "Channel: English - ASIO Focusrite")

    Export all types. Use `export type` for type-only exports where possible.
  </action>

  <name>Task 2: Extend Zod config schemas for audio channels and sources</name>
  <files>
    sidecar/src/config/schema.ts

  </files>
  <action>
    **Extend schema.ts with audio configuration schemas:**

    Add new Zod schemas BELOW the existing CertificateSchema, BEFORE ConfigSchema:

    - `LevelMeteringSchema` = z.object:
      - `intervalMs`: z.number().int().min(10).max(1000).default(100) -- level update interval (100ms = 10/sec, 33ms = 30/sec)

    - `SourceAssignmentSchema` = z.object:
      - `sourceId`: z.string()
      - `selectedChannels`: z.array(z.number().int().min(0)).min(1) -- at least one channel selected
      - `gain`: z.number().min(0).max(2).default(1.0)
      - `muted`: z.boolean().default(false)
      - `delayMs`: z.number().min(0).max(5000).default(0) -- max 5 seconds delay

    - `ChannelSchema` = z.object:
      - `id`: z.string().uuid()
      - `name`: z.string().min(1).max(100)
      - `sources`: z.array(SourceAssignmentSchema).default([])
      - `outputFormat`: z.enum(["mono", "stereo"]).default("mono")
      - `autoStart`: z.boolean().default(true)

    - `PipelineRecoverySchema` = z.object:
      - `autoRestart`: z.boolean().default(true)
      - `maxRestartAttempts`: z.number().int().min(0).max(20).default(5)
      - `restartDelayMs`: z.number().int().min(500).max(30000).default(2000)
      - `drainTimeoutMs`: z.number().int().min(0).max(5000).default(500) -- graceful shutdown drain period

    - `DiscoveryCacheSchema` = z.object:
      - `enabled`: z.boolean().default(true) -- persist discovered streams between restarts
      - `devicePollIntervalMs`: z.number().int().min(1000).max(30000).default(5000) -- how often to re-enumerate local devices

    - `AudioSchema` = z.object:
      - `channels`: z.array(ChannelSchema).default([])
      - `levelMetering`: LevelMeteringSchema.default(...)
      - `pipelineRecovery`: PipelineRecoverySchema.default(...)
      - `discoveryCache`: DiscoveryCacheSchema.default(...)

    **Update ConfigSchema:** Add `audio: AudioSchema.default(...)` to the existing ConfigSchema object. This means ConfigSchema now has: server, network, certificate, audio.

    **Update defaults.ts:** No changes needed -- the default config is derived from `ConfigSchema.parse({})` which auto-applies all defaults.

    **Important:** Use the same Zod 4.x factory default pattern as existing schemas: `.default(() => SubSchema.parse({}))` for nested objects.

    **Do NOT change** the existing ServerSchema, NetworkSchema, CertificateSchema, or their defaults. Only ADD the new audio schemas and extend ConfigSchema.
  </action>
