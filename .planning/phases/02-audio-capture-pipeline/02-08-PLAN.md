  <action>
    Create `channel-manager.ts` that exports a `ChannelManager` class -- the central orchestrator for app channels.

    **Imports:**
    - `crypto` from `node:crypto`
    - `EventEmitter` from `node:events`
    - `AppChannel, SourceAssignment, ChannelStatus` from `./channel-types`
    - `PipelineManager` from `../pipeline/pipeline-manager`
    - `PipelineConfig` from `../pipeline/pipeline-types`
    - `SourceRegistry` from `../sources/source-registry`
    - `AES67Source, LocalDeviceSource, DiscoveredSource` from `../sources/source-types`
    - `LevelMonitor` from `../monitor/level-monitor`
    - `ResourceMonitor` from `../monitor/resource-monitor`
    - `EventLogger` from `../monitor/event-logger`
    - `ConfigStore` from `../../config/store`
    - `logger` from `../../utils/logger`

    **ChannelManager extends EventEmitter with events:**
    - `"channel-created"` -> (channel: AppChannel)
    - `"channel-updated"` -> (channel: AppChannel)
    - `"channel-removed"` -> (channelId: string)
    - `"channel-state-changed"` -> (channelId: string, status: ChannelStatus)

    **Internal state:**
    - `channels: Map<string, AppChannel>` -- all app channels keyed by channel ID
    - `channelPipelines: Map<string, Map<string, string>>` -- channelId -> Map<sourceAssignmentIndex, pipelineId>
    - Dependencies injected via constructor

    **Constructor:**
    ```typescript
    constructor(
      private pipelineManager: PipelineManager,
      private sourceRegistry: SourceRegistry,
      private levelMonitor: LevelMonitor,
      private resourceMonitor: ResourceMonitor,
      private eventLogger: EventLogger,
      private configStore: ConfigStore,
    )
    ```
    - Wire up PipelineManager event listeners:
      - On "pipeline-state-change": update channel status (aggregate from all pipelines)
      - On "pipeline-levels": forward to LevelMonitor.handleLevels()
      - On "pipeline-error": log via EventLogger, update channel status
    - Load channels from config store (configStore.get().audio.channels)
    - Populate channels map (but do NOT auto-start yet -- that's done explicitly)

    **Method: `createChannel(name: string, outputFormat?: "mono" | "stereo"): AppChannel`**
    - Generate UUID for channel ID
    - Create AppChannel object with default values (empty sources, autoStart true, status "stopped")
    - Add to channels map
    - Persist to config store (update audio.channels array)
    - Log event via EventLogger: type "info", message "Channel created"
    - Emit "channel-created"
    - Return the channel

    **Method: `removeChannel(channelId: string): Promise<void>`**
    - Stop all pipelines for this channel first
    - Remove from channels map and channelPipelines map
    - Clear events in EventLogger for this channel? No -- keep logs for history (admin may review)
    - Remove from config store
    - Emit "channel-removed"

    **Method: `updateChannel(channelId: string, updates: Partial<Pick<AppChannel, "name" | "outputFormat" | "autoStart">>): AppChannel`**
    - Apply updates to channel
    - Persist to config store
    - Emit "channel-updated"
    - Return updated channel

    **Method: `addSource(channelId: string, assignment: SourceAssignment): AppChannel`**
    - Validate the source exists in sourceRegistry (by sourceId). If not found, throw "Source not found: {sourceId}"
    - Validate selectedChannels are within range of the source's channelCount
    - Add assignment to channel's sources array
    - If channel is currently streaming, start a new pipeline for this source assignment (hot-add)
    - Persist to config store
    - Log event: "Source added: {sourceName}"
    - Emit "channel-updated"
    - Return updated channel

    **Method: `removeSource(channelId: string, sourceIndex: number): Promise<AppChannel>`**
    - Validate index is within bounds
    - If pipeline exists for this source, stop it
    - Remove from sources array
    - Persist to config store
    - Log event: "Source removed"
    - Emit "channel-updated"
    - Return updated channel

    **Method: `updateSource(channelId: string, sourceIndex: number, updates: Partial<SourceAssignment>): AppChannel`**
    - Apply updates (gain, mute, delay, selectedChannels)
    - If selectedChannels changed and pipeline is running: restart pipeline (instant cut per CONTEXT.md discretion decision)
    - If only gain/mute/delay changed: these will be handled in Phase 3 (mixing), for now persist only
    - Persist to config store
    - Emit "channel-updated"

    **Method: `startChannel(channelId: string): Promise<void>`**
    - For each source assignment in the channel:
      - Build PipelineConfig from source assignment + source registry info
      - Create pipeline via pipelineManager.createPipeline(config)
      - Start pipeline via pipelineManager.startPipeline(pipelineId)
      - Track pipeline in resourceMonitor.trackPipeline(pipelineId, pid)
      - Store mapping in channelPipelines
    - Update channel status to "starting"
    - Log event: "Channel starting with {N} sources"

    **Method: `stopChannel(channelId: string): Promise<void>`**
    - For each pipeline in channelPipelines for this channel:
      - Stop pipeline via pipelineManager.stopPipeline(pipelineId)
      - Untrack from resourceMonitor
      - Clear from levelMonitor
    - Clear channelPipelines for this channel
    - Update channel status to "stopped"
    - Log event: "Channel stopped"

    **Method: `startAll(): Promise<void>`**
    - For each channel where autoStart === true:
      - Call startChannel(channelId)
    - Log summary: "Auto-started {N} channels"
    - This is called from sidecar/src/index.ts on app launch

    **Method: `stopAll(): Promise<void>`**
    - Stop all channels
    - Called during graceful shutdown

    **Method: `getChannel(channelId: string): AppChannel | undefined`**
    **Method: `getAllChannels(): AppChannel[]`**
    **Method: `getChannelPipelineIds(channelId: string): string[]`** -- returns pipeline IDs for a channel

    **Private: `buildPipelineConfigFromAssignment(assignment: SourceAssignment): PipelineConfig`**
    - Look up source in sourceRegistry by sourceId
    - If source.type === "aes67": build aes67Config from AES67Source fields + assignment.selectedChannels
    - If source.type === "local": build localConfig from LocalDeviceSource fields + assignment.selectedChannels
    - Set levelIntervalMs from configStore audio.levelMetering.intervalMs
    - Set label from channel name + source name
    - Return PipelineConfig

    **Private: `aggregateChannelStatus(channelId: string): ChannelStatus`**
    - Get all pipeline IDs for this channel
    - If no pipelines: "stopped"
    - If any pipeline is "crashed" or "error": "error"
    - If all pipelines are "streaming": "streaming"
    - If any pipeline is "initializing"/"connecting"/"buffering": "starting"
    - Otherwise: "stopped"

    **Private: `persistChannels(): void`**
    - Convert channels map to array of channel objects (matching ChannelSchema)
    - Call configStore.update({ audio: { channels: channelArray } })

    **Important design decisions:**
    - Source switching uses instant cut (stop old pipeline, start new) per CONTEXT.md Claude's discretion decision
    - Multiple sources per channel each get their own pipeline. Phase 2 does NOT mix them -- that's Phase 3. For now, each pipeline runs independently and level data is available per-pipeline.
    - Channel status is derived from pipeline states (aggregate)
    - Pipeline creation validates source existence -- if a configured source is not found (stale config), log warning and skip that source
