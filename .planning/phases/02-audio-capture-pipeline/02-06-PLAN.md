<objective>
Build the source registry (unified store for all discovered audio sources) and the discovery manager (coordinator that ties SAP, mDNS, and device enumeration together).

Purpose: The source registry is the single source of truth for all available audio inputs. The discovery manager wires SAP listener events, device enumerator events, and mDNS (bonjour-service already in deps) into the registry. Sources persist to a JSON cache file so they're instantly available on restart.

Output: `source-registry.ts` provides a typed in-memory store with persistence. `discovery-manager.ts` coordinates all discovery mechanisms and manages their lifecycle.
</objective>

  <name>Task 1: Create SourceRegistry with persistence</name>
  <files>sidecar/src/audio/sources/source-registry.ts</files>
  <action>
    Create `source-registry.ts` that exports a `SourceRegistry` class -- the unified store for all discovered audio sources.

    **Imports:**
    - `EventEmitter` from `node:events`
    - `fs` from `node:fs`
    - `path` from `node:path`
    - `DiscoveredSource, AES67Source, LocalDeviceSource, SourceStatus` from `./source-types`
    - `logger` from `../../utils/logger`

    **SourceRegistry extends EventEmitter with events:**
    - `"source-added"` -> (source: DiscoveredSource)
    - `"source-updated"` -> (source: DiscoveredSource) -- status or metadata changed
    - `"source-removed"` -> (sourceId: string)
    - `"sources-changed"` -> () -- any change to the registry (for UI refresh)

    **Internal state:**
    - `sources: Map<string, DiscoveredSource>` -- keyed by source ID
    - `cacheFilePath: string`

    **Constructor:** `constructor(basePath: string)`
    - Set cacheFilePath to `path.join(basePath, "discovered-sources.json")`
    - Load cached sources from disk (if file exists)
    - Mark all loaded sources as "verifying" status (they'll be confirmed by live discovery)

    **Method: `addOrUpdate(source: DiscoveredSource): void`**
    - If source.id exists in map:
      - Deep compare relevant fields. If changed, update and emit "source-updated"
      - Always update `lastSeenAt` timestamp
    - If source.id NOT in map:
      - Add to map, emit "source-added"
    - Emit "sources-changed"
    - Persist to disk (debounced, see below)

    **Method: `updateStatus(sourceId: string, status: SourceStatus): void`**
    - Find source, update status, emit "source-updated" and "sources-changed"
    - Persist to disk (debounced)

    **Method: `markUnavailable(sourceId: string): void`**
    - Set status to "unavailable" (NOT removed -- CONTEXT.md: offline streams show as unavailable, not removed)
    - Emit "source-updated" and "sources-changed"

    **Method: `remove(sourceId: string): void`**
    - Only for permanent removal (e.g., SAP deletion packet)
    - Delete from map, emit "source-removed" and "sources-changed"
    - Persist to disk

    **Method: `getAll(): DiscoveredSource[]`**
    - Return Array.from(sources.values()), sorted by type then name

    **Method: `getById(sourceId: string): DiscoveredSource | undefined`**
    - Return sources.get(sourceId)

    **Method: `getByType(type: "aes67" | "local"): DiscoveredSource[]`**
    - Filter sources by type discriminator

    **Method: `getAvailable(): DiscoveredSource[]`**
    - Filter sources where status is "available" or "in-use"

    **Persistence (private):**
    - `persistToDisk()`: Write sources map to cacheFilePath as JSON. Debounce with 2-second delay to avoid excessive disk writes during rapid discovery.
    - `loadFromDisk()`: Read cacheFilePath, parse JSON, validate entries, populate sources map. On parse error, log warning and start with empty map.
    - Use `setTimeout` for debouncing, clear previous timeout on each persist call

    **Important:**
    - Source IDs must be deterministic: for AES67, use `aes67:${originAddress}:${originSessionId}`; for local devices, use `local:${api}:${deviceId}`. This ensures the same physical source always gets the same ID across restarts.
    - The cache file stores serialized DiscoveredSource objects. On load, validate each entry has required fields.
    - Do NOT remove sources automatically on timeout. Only mark as unavailable. Admin explicitly removes stale sources (or they just stay grayed out).
  </action>
  <verify>
    `npx tsc --noEmit` passes. SourceRegistry exports with all documented methods. Verify persistence: sources round-trip through JSON serialization. Verify ID generation is deterministic for both AES67 and local sources. Verify status transitions: verifying -> available, available -> unavailable.
  </verify>
  <done>
    SourceRegistry provides unified storage for all discovered audio sources with events for UI updates. Persistence via JSON cache file with debounced writes. Sources survive restarts and are re-verified by live discovery. Unavailable sources are grayed out, not removed.

  <name>Task 2: Create DiscoveryManager coordinator</name>
  <files>sidecar/src/audio/discovery/discovery-manager.ts</files>
  <action>
    Create `discovery-manager.ts` that exports a `DiscoveryManager` class coordinating all discovery mechanisms.

    **Imports:**
    - `SapListener` from `./sap-listener`
    - `DeviceEnumerator, EnumeratedDevice` from `./device-enumerator`
    - `SourceRegistry` from `../sources/source-registry`
    - `AES67Source, LocalDeviceSource, SourceStatus` from `../sources/source-types`
    - `Aes67SdpInfo` from `./sdp-parser`
    - `logger` from `../../utils/logger`
    - bonjour-service for mDNS discovery (already in project deps)

    **DiscoveryManager:**
    - `private sapListener: SapListener`
    - `private deviceEnumerator: DeviceEnumerator`
    - `private sourceRegistry: SourceRegistry`
    - `private mdnsBrowser: any` (from bonjour-service, for RAVENNA device discovery)
    - `private running: boolean`

    **Constructor:** `constructor(sourceRegistry: SourceRegistry, options: { networkInterfaceAddress?: string, devicePollIntervalMs?: number })`
    - Create SapListener instance
    - Create DeviceEnumerator instance with pollIntervalMs
    - Store sourceRegistry reference
    - Wire up event handlers (see below)

    **Method: `start(): Promise<void>`**
    1. Start SAP listener (with network interface address if provided)
    2. Start mDNS browser for RAVENNA services:
       - Browse for `_ravenna_session._sub._rtsp._tcp` service type
       - On service-up: extract service info, try to discover AES67 SDP (RAVENNA exposes SDP via RTSP, but SAP is the primary mechanism). Log discovered RAVENNA device for now; full RTSP SDP fetch is a future enhancement.
    3. Run initial device enumeration
    4. Start device polling
    5. Log "Discovery manager started: SAP + mDNS + device polling"

    **Method: `stop(): Promise<void>`**
    1. Stop SAP listener
    2. Stop mDNS browser
    3. Stop device polling
    4. Log "Discovery manager stopped"

    **Event wiring (private):**

    SAP listener events -> source registry:
    - On "stream-discovered": Convert Aes67SdpInfo to AES67Source type:
      - `id: "aes67:${info.originAddress}:${info.originSessionId}"`
      - `type: "aes67"`
      - `name: info.sessionName`
      - `description: info.description`
      - `multicastAddress: info.multicastAddress`
      - `port: info.port`
      - `sampleRate: info.sampleRate`
      - `bitDepth: info.bitDepth`
      - `channelCount: info.channelCount`
      - `payloadType: info.payloadType`
      - `originAddress: info.originAddress`
      - `channelLabels: info.channelLabels`
      - `status: "available"`
      - `lastSeenAt: Date.now()`
      - `discoveredAt: Date.now()`
      - Call `sourceRegistry.addOrUpdate(aes67Source)`

    - On "stream-updated": Same conversion, call `sourceRegistry.addOrUpdate()` (it handles diff)

    - On "stream-removed": Call `sourceRegistry.remove("aes67:${originAddress}:${originSessionId}")`

    Device enumerator events -> source registry:
    - On "device-added": Convert EnumeratedDevice to LocalDeviceSource type:
      - `id: "local:${device.api}:${device.deviceId}"`
      - `type: "local"`
      - `name: "${device.name} (${apiDisplayName(device.api)})"` -- e.g., "Focusrite USB (WASAPI)" or "Focusrite USB (ASIO)"
      - `api: device.api`
      - `deviceId: device.deviceId`
      - `sampleRate: device.sampleRate`
      - `bitDepth: device.bitDepth`
      - `channelCount: device.channelCount`
      - `isLoopback: device.isLoopback`
      - `status: "available"`
      - `lastSeenAt: Date.now()`
      - Call `sourceRegistry.addOrUpdate(localSource)`

    - On "device-removed": Call `sourceRegistry.markUnavailable("local:${api}:${deviceId}")` -- mark unavailable, not removed (CONTEXT.md: hot-unplug recovery needs to track the device)

    - On "enumeration-complete": For each device in currentDevices, update lastSeenAt. For any local sources in registry that are NOT in the new enumeration list, mark unavailable.

    **Helper: `apiDisplayName(api: string): string`**
    - "wasapi2" -> "WASAPI"
    - "asio" -> "ASIO"
    - "directsound" -> "DirectSound"

    **Method: `getSourceRegistry(): SourceRegistry`**
    - Return the registry reference (for other managers to subscribe to events)

    **Important:**
    - CONTEXT.md specifies mDNS/Bonjour for discovery alongside SAP. bonjour-service is already in deps.
    - The mDNS browser for RAVENNA is supplementary -- SAP is the primary AES67 discovery mechanism. If mDNS finds a RAVENNA device but SAP hasn't announced it yet, log it but don't create a source (we need SDP info from SAP to build pipelines).
    - Source IDs are deterministic across restarts: same device always maps to same ID.
