# Plan 02-10: Rebuild Sidecar Binary

## Goal

Rebuild the sidecar pkg binary so it includes all Phase 2 audio subsystem code (AudioSubsystem, audio WebSocket handlers, discovery, channels, pipelines, monitors).

## Root Cause

The compiled binary (`src-tauri/binaries/server-x86_64-pc-windows-msvc.exe`) was built from Phase 1 source. The `sidecar/dist/` directory and binary do not contain any Phase 2 code. Source code is correct -- only compilation is needed.

## Tasks

### Task 1: Rebuild sidecar binary

**Steps:**
1. Run `npm run build` from the `sidecar/` directory (compiles TypeScript to `dist/`, then runs pkg to produce binary)
2. Verify `dist/audio/` directory exists after compilation
3. Verify `dist/index.js` contains AudioSubsystem import and creation
4. Verify `dist/ws/handler.js` contains handleAudioMessage function
5. Verify binary in `src-tauri/binaries/` is updated (newer timestamp)

**Verification:**
- `npx tsc --noEmit` passes (no type errors)
- `dist/audio/` directory exists with all audio modules
- Binary file size may increase (more code compiled in)
