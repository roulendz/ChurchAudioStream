<objective>
Implement AES67 stream discovery via SAP (Session Announcement Protocol) multicast listener and SDP (Session Description Protocol) parser.

Purpose: AES67/Dante streams announce themselves via SAP packets containing SDP descriptions. This is the only way to discover available streams on the network (CONTEXT.md: no manual multicast entry, discovery only). The SAP listener runs continuously from app startup, and discovered streams persist between restarts.

Output: `sap-listener.ts` listens on the SAP multicast group and emits discovery events. `sdp-parser.ts` extracts AES67-specific information from SDP payloads using the sdp-transform library.
</objective>

  <name>Task 1: Create SDP parser for AES67 streams</name>
  <files>sidecar/src/audio/discovery/sdp-parser.ts</files>
  <action>
    Create `sdp-parser.ts` that wraps sdp-transform for AES67-specific SDP extraction.

    **Install note:** sdp-transform should already be installed from Plan 01. If not, this module still compiles -- the import will just fail at runtime.

    **Types (exported):**

    `Aes67SdpInfo`:
    - `sessionName: string` -- from SDP `s=` field (stream name set in Dante Controller)
    - `description: string` -- from SDP `i=` field (information line), empty string if absent
    - `multicastAddress: string` -- from SDP `c=IN IP4 {address}/{ttl}` connection line
    - `port: number` -- from SDP media line `m=audio {port} RTP/AVP {pt}`
    - `sampleRate: number` -- from SDP `a=rtpmap:{pt} L{bitDepth}/{sampleRate}/{channels}`
    - `channelCount: number` -- from SDP rtpmap line (channel count parameter, default 1 if omitted)
    - `bitDepth: number` -- 16 or 24, from SDP rtpmap encoding name (L16 or L24)
    - `payloadType: number` -- RTP payload type from media line
    - `channelLabels: string[]` -- from SDP `a=label:` attributes, one per channel; empty array if none
    - `originAddress: string` -- from SDP `o=` line, origin IP address
    - `originSessionId: string` -- from SDP `o=` line, session ID (used as unique identifier together with origin address)

    **Exported function: `parseAes67Sdp(sdpContent: string): Aes67SdpInfo | null`**

    Implementation:
    1. Call `parse(sdpContent)` from sdp-transform to get the parsed SDP object
    2. Extract `sessionName` from `sdp.name` (the `s=` line)
    3. Extract `description` from `sdp.description` (the `i=` line), default empty string
    4. Extract origin from `sdp.origin`: `originAddress = sdp.origin.address`, `originSessionId = sdp.origin.sessionId`
    5. Find the audio media block: `sdp.media.find(m => m.type === 'audio')`
       - If no audio media found, return null (not an audio stream)
    6. Extract `port` from the audio media block: `media.port`
    7. Extract multicast address: Check `media.connection?.ip` first (media-level), fall back to `sdp.connection?.ip` (session-level)
    8. Extract payload type: `media.payloads` (string of payload types, take first one for AES67 which typically has one)
    9. Find the rtpmap attribute for this payload type in `media.rtp`: look for entry where `payload` matches
       - Parse encoding name (e.g., "L24", "L16") to extract bitDepth
       - Extract `sampleRate` from `rate` field
       - Extract `channelCount` from `encoding` field (sdp-transform parses channels as the encoding parameter). If not present, default to 1.
    10. Extract channel labels: filter `media.invalid` (sdp-transform puts unrecognized attributes here) for `a=label:` attributes. Parse the value after "label:". Return as array.
    11. If critical fields are missing (no audio media, no multicast address, no rtpmap), return null
    12. Return the Aes67SdpInfo object

    **Edge cases:**
    - Some SDP uses `a=rtpmap:98 L24/48000/2` (with channels), some use `a=rtpmap:98 L24/48000` (mono implied)
    - Channel labels may not be present at all (most Dante devices don't send them)
    - The connection line may have TTL: `c=IN IP4 239.69.0.121/32` -- sdp-transform handles this
    - Multiple media blocks: only process the first `audio` type

    **Do NOT** hand-roll SDP parsing. Use sdp-transform for the heavy lifting, then extract AES67-specific fields.

<task type="auto">
  <name>Task 2: Create SAP multicast listener</name>
  <files>sidecar/src/audio/discovery/sap-listener.ts</files>
  <action>
    Create `sap-listener.ts` that exports a `SapListener` class for receiving SAP multicast announcements on 224.2.127.254:9875.

    **Imports:**
    - `dgram` from `node:dgram`
    - `EventEmitter` from `node:events`
    - `parseAes67Sdp, Aes67SdpInfo` from `./sdp-parser`
    - `logger` from `../../utils/logger`

    **Constants:**
    - `SAP_MULTICAST_ADDRESS = "224.2.127.254"`
    - `SAP_PORT = 9875`
    - `SAP_HEADER_MIN_LENGTH = 8` (minimum SAP header size in bytes)

    **SapListener extends EventEmitter with events:**
    - `"stream-discovered"` -> (info: Aes67SdpInfo & { sapHash: number })
    - `"stream-updated"` -> (info: Aes67SdpInfo & { sapHash: number }) -- same stream, new SDP version
    - `"stream-removed"` -> (sapHash: number, originAddress: string)
    - `"error"` -> (error: Error)

    **Internal state:**
    - `socket: dgram.Socket | null`
    - `knownStreams: Map<string, { sdpInfo: Aes67SdpInfo, sapHash: number, lastSeen: number }>` -- keyed by `${originAddress}:${originSessionId}`
    - `running: boolean`

    **Constructor:** `constructor()` -- no args, does not start listening

    **Method: `start(networkInterfaceAddress?: string): void`**
    - If already running, return
    - Create UDP4 socket with `reuseAddr: true`
    - Bind to SAP_PORT
    - On bind success: call `socket.addMembership(SAP_MULTICAST_ADDRESS, networkInterfaceAddress)` -- if networkInterfaceAddress provided, bind to that interface; otherwise let OS choose (note RESEARCH Pitfall 1: Windows may not respect this, but it's still correct to try)
    - Set `socket.setMulticastTTL(32)` (standard for SAP)
    - Attach message handler (see below)
    - Attach error handler: emit "error" event, log
    - Set running = true
    - Log "SAP listener started on {SAP_MULTICAST_ADDRESS}:{SAP_PORT}"

    **Method: `stop(): void`**
    - If not running, return
    - Try `socket.dropMembership(SAP_MULTICAST_ADDRESS)`
    - Close socket
    - Set running = false, socket = null
    - Log "SAP listener stopped"

    **Method: `getKnownStreams(): Map<string, { sdpInfo: Aes67SdpInfo, sapHash: number, lastSeen: number }>`**
    - Return a copy of knownStreams for persistence

    **Method: `loadCachedStreams(cached: Map<string, { sdpInfo: Aes67SdpInfo, sapHash: number, lastSeen: number }>): void`**
    - Load previously discovered streams on startup (CONTEXT.md: persist between restarts)
    - Set each stream's status context (the source registry will handle "verifying" status)

    **Private: `parseSapPacket(buffer: Buffer)`**
    - If buffer.length < SAP_HEADER_MIN_LENGTH, return null
    - Parse first byte: version = bits 5-7, messageType bit 2 (0 = announcement, 1 = deletion), addressType bit 4 (0 = IPv4, 1 = IPv6)
    - If version !== 1, return null (only SAP v1)
    - Parse authentication length from byte 1 (number of 32-bit auth words)
    - Parse SAP hash from bytes 2-3 (big-endian uint16)
    - Parse origin address from bytes 4-7 (IPv4, 4 bytes -> dotted decimal)
    - Skip authentication data: authLength * 4 bytes after the 8-byte header
    - Find MIME type separator: scan for null byte after auth data. The content type string (usually "application/sdp\0") precedes the SDP.
    - Extract SDP content: everything after the null-terminated content type string
    - Return `{ version, isAnnouncement, sapHash, originAddress, sdpContent }`

    **Private: message handler `handleSapMessage(buffer: Buffer, rinfo: dgram.RemoteInfo)`**
    - Parse SAP packet
    - If null (malformed), return silently
    - If deletion packet (isAnnouncement === false):
      - Find and remove stream by sapHash + originAddress
      - Emit "stream-removed"
      - Log "AES67 stream removed: {sapHash}"
      - Return
    - If announcement packet:
      - Parse SDP via `parseAes67Sdp(sdpContent)`
      - If null (not an audio stream), return
      - Build unique key: `${sdpInfo.originAddress}:${sdpInfo.originSessionId}`
      - If key already in knownStreams:
        - Update lastSeen timestamp
        - Check if SDP changed (compare relevant fields). If changed, emit "stream-updated"
      - If key NOT in knownStreams:
        - Add to knownStreams
        - Emit "stream-discovered"
      - Log at debug level (SAP can be chatty with 300s intervals per Pitfall 4)

    **Important per RESEARCH:**
    - SAP announcements can take up to 300 seconds (Pitfall 4). The app should load cached streams on startup and update as SAP packets arrive.
    - SAP hash is NOT guaranteed unique across different origins. Use originAddress + originSessionId as the unique key.
    - Do NOT join any multicast audio groups here -- SAP is discovery only. Audio multicast join happens in the pipeline (join on demand per CONTEXT.md).
  </action>
  <verify>
    `npx tsc --noEmit` passes. SapListener class exports with start(), stop(), getKnownStreams(), loadCachedStreams() methods. Verify SAP packet parser handles the standard 8-byte header + MIME type + SDP layout. Verify event emissions for discovered/updated/removed.
  </verify>
